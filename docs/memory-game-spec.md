# 말로 열기 게임 기획/구현 문서

## 1. 개요

`말로 열기`는 그림을 보고 해당 그림이 속하는 **분류 카테고리**를 말해 맞히는 게임이다.  
사용자는 사물 이름 자체를 말하는 것이 아니라, `과일 / 동물 / 탈것` 중 올바른 분류를 발화해야 한다.

현재 구현은 **브라우저 SpeechRecognition 기반 자동 듣기 게임**이다.

---

## 2. 게임 목표

- 그림을 보고 올바른 카테고리를 말한다.
- 제한 시간 안에 최대한 많은 카드를 연다.
- 카테고리 개념 분류와 음성 발화를 동시에 훈련한다.

핵심 입력은 다음 3개뿐이다.

- `과일`
- `동물`
- `탈것`

- 현재 제한 시간: `10초`

### 보통

- 카테고리별 카드 수: `3장`
- 전체 카드 수: `9장`
- 현재 제한 시간: `7초`

### 어려움

- 카테고리별 카드 수: `4장`
- 전체 카드 수: `12장`

카드 덱은 난이도 선택 시:

1. 카테고리별로 필요한 개수만 랜덤 추출
2. 전체를 다시 셔플

하는 방식으로 구성된다.

## 5. 입력 방식

말로 열기는 [MemoryFlipGame.tsx](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/components/lingo/MemoryFlipGame.tsx) 안에서 `window.SpeechRecognition` 또는 `window.webkitSpeechRecognition`을 사용한다.

즉:

- 서버 STT 아님
- Whisper 아님
- 브라우저 기본 음성 인식 기반

따라서 실사용 기준으론 **Chrome 계열 브라우저**가 가장 적합하다.

## 6. 분류어 판정 방식

현재 게임 컨셉은 “그림 이름 맞히기”가 아니라 “카테고리 맞히기”이므로, 정답 처리 기준은 분류어다.

### 인정되는 발화

- `과일`
- `과일이요`
- `과일이에요`
- `과일입니다`

- `동물`
- `동물이요`
- `동물이에요`
- `동물입니다`

- `탈것`
- `탈것이요`
- `탈것이에요`
- `탈것입니다`
- `탈거` 등 일부 흔한 오인식 변형

### 인정되지 않는 발화

- 사과
- 바나나
- 토끼
- 비행기

즉 그림 이름은 정답이 아니고, **카테고리만 정답**이다.

---

## 7. 라운드 구조

현재 말로 열기의 흐름은 다음과 같다.

1. 난이도 선택
2. 덱 생성
3. 현재 목표 카드 선택
4. 자동 듣기 시작
5. 제한 시간 안에 분류어를 말함
6. 정답이면 바로 다음 카드
7. 시간이 끝나면 실패 처리 후 다음 카드
8. 모든 카드가 열리면 결과 리포트 표시

중요한 점:

- 애매한 인식값은 바로 실패 처리하지 않음
- 오답을 말했다고 즉시 다음 카드로 넘기지 않음
- 시간 초과일 때만 실패 확정 후 다음 카드로 이동

즉 사용자는 제한 시간 안에 **여러 번 다시 말할 기회**를 가진다.

---

## 8. 카드 전환 로직

이전에는 카드 index 기반과 음성 carry-over 문제로 카드가 꼬이는 이슈가 있었지만, 현재는 다음 원칙으로 정리되어 있다.

- 현재 목표 카드는 `targetWord` 기준으로 직접 추적
- 카드가 바뀌면 `heardText`와 안내 문구를 초기화
- 직전 인식값이 다음 카드에 바로 먹지 않도록 짧은 무시 구간 적용
- 정답/시간초과 처리 중에는 transition lock 사용

즉 한 번 정답이 나오면:

1. 현재 카드만 열린다
2. 다음 목표 카드가 정해진다
3. 이전 인식값은 제거된다

---

## 9. 시간 제한 처리

제한 시간은 난이도별로 다르며, `timeLeft`로 관리된다.

시간이 줄어드는 동안:

- 메인 카드 상단에 남은 시간 표시
- `Round Timer` 바 표시

시간이 0이 되면:

- 현재 카드 실패 처리
- `wrongCount` 증가
- 다음 카드로 자동 이동

정답은 시간이 끝나기 전에 말하면 **즉시 처리**된다.

---

## 10. 점수 측정 방식

현재 말로 열기의 성과 지표는 다음과 같다.

- `solvedCount`: 맞힌 카드 수
- `wrongCount`: 시간 초과 등 실패 카드 수
- `attemptCount`: 시도 횟수
- `accuracy`: 맞힌 카드 비율 기반 정확도
- `score`: 내부 점수 지표

구체적 체감은 “점수 게임”보다 “카드를 얼마나 열었는가”에 더 가깝다.

오른쪽 `Session Progress`에서는:

- 열린 카드 수
- 전체 카드 수
- 현재 진행률

을 보여준다.

현재 말로 열기 메인 화면은 다음 구조다.

### 우측 사이드 패널

## 12. 자동 듣기 UX

말로 열기는 현재 마이크 버튼을 누르는 게임이 아니라, **계속 듣는 상태에서 바로 말하는 게임**으로 설계되어 있다.

사용자 체감 강화를 위해 다음 요소를 둔다.

- `자동 듣기 활성화`
- `녹음 중 / 녹음 대기`
- `Listening Stream`
- 정답 시 즉시 다음 카드

즉 조작보다 **즉시 반응성**이 중요하다.

---

## 13. 결과 리포트

결과 화면은 공통 결과 모달 셸을 사용한다.

주요 표시 항목:

- 훈련 완료
- 성공률
- 정확도
- 걸린 시간

그리고 사용자 요구에 따라:

- `Session Report Index`는 제거
- 한글 라벨로 통일
- `메인 화면으로 돌아가기` 버튼 포함

---

## 14. 관리자/디버그 요소

관리자/로컬 환경에서만 헤더에 분류 강제 버튼이 보인다.

- 과일
- 동물
- 탈것

이는 음성 없이 빠르게 흐름을 테스트하기 위한 디버그 도구다.

---

## 15. 장점

- 규칙이 단순해 초반 진입이 쉬움
- 카테고리 개념 훈련과 음성 입력을 결합
- 제한 시간이 있어 게임성이 생김
- 버튼 입력 없이 자동 듣기로 흐름이 빠름

---

## 16. 현재 구현상 주의점

- 브라우저 음성 인식 품질은 Chrome 계열에서 가장 안정적이다.
- 같은 카테고리 카드가 연속으로 나올 경우 carry-over 방지가 중요하다.
- 그림 이름이 아니라 분류어만 정답이므로, 사용자 안내가 명확해야 한다.
- 자동 듣기 UX가 약하면 “지금 듣고 있나?”라는 불안이 생길 수 있다.

---

## 17. 향후 개선 제안

- 지원 브라우저 감지 및 안내
- `Listening Stream`에 확정 분류어를 더 선명하게 표시
- 카드별 시간 초과 애니메이션 강화
- 분류어 발화 예시(예: “과일이에요”)를 시작 모달에 함께 제시

---
