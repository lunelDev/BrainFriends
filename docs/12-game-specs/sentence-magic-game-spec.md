# 문장 마법 게임 기획/구현 문서

## 1. 개요

`문장 마법`은 사용자가 제시된 목표 문장을 보고 발화한 뒤, 발화 정확도에 따라 전투 승패를 결정하는 음성 훈련 게임이다.  
게임 내부에는 두 가지 모드가 있다.

- `문장 만들기`
- `잰말 놀이`

즉 하나의 게임 안에서

- 문장 구성 능력
- 발화 정확도
- 빠른 조음/유창성

을 함께 다룬다.

---

## 2. 게임 목표

- 목표 문장을 정확하게 발화한다.
- 기준 정확도 이상이면 상대 HP를 깎는다.
- 내 HP가 0이 되기 전에 상대를 먼저 쓰러뜨린다.

즉 문장 마법은 일반 점수 게임이 아니라 **전투형 정확도 판정 게임**에 가깝다.

---

## 3. 모드 구성

데이터는 [sentenceGameData.ts](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/data/sentenceGameData.ts)에 정의되어 있다.

### 3-1. 문장 만들기

- 사용자가 단어 조각을 보고 올바른 문장으로 말하는 모드
- 정답 문장과 유사도를 비교
- 문장 구성 및 순서화 훈련 목적

데이터:

- `SENTENCE_EXAMPLE_QUESTIONS`
- 현재 총 `22개`

### 3-2. 잰말 놀이

- 제시된 문장을 그대로 또렷하게 말하는 모드
- 빠른 조음과 발화 정확도 훈련 목적

데이터:

- `TONGUE_TWISTER_QUESTIONS`
- threshold 그룹별로 나뉜다

---

## 4. 난이도 구성

문장 마법의 난이도는 `잰말 놀이`에서 threshold로 표현된다.

### 하

- threshold: `55`

### 중

- threshold: `65`

### 상

- threshold: `75`

이 값은 [TONGUE_TWISTER_THRESHOLDS](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/data/sentenceGameData.ts) 에 정의되어 있으며,  
판정 기준과 문제군 선택에 동시에 사용된다.

문제군은 다음처럼 나뉜다.

- `55`: 첫 20문항
- `65`: 다음 20문항
- `75`: 나머지 문항

즉 난이도가 올라갈수록:

- 더 어려운 문장이 나오고
- 통과 기준도 높아진다.

---

## 5. 입력 방식

문장 마법은 **서버 STT 기반**이다.

현재 구조:

1. `MediaRecorder`로 음성 녹음
2. Blob 생성
3. `/api/proxy/stt`로 업로드
4. 반환된 transcript를 목표 문장과 비교

사용 파일:

- [SentenceMagicGame.tsx](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/components/lingo/SentenceMagicGame.tsx)
- [route.ts](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/app/api/proxy/stt/route.ts)

즉 문장 마법은 브라우저 SpeechRecognition이 아니라 **Whisper 계열 STT 프록시**를 통한 전사 게임이다.

---

## 6. 정확도 계산 방식

정확도는 `getAccuracyScore(heard, answer)`에서 계산한다.

흐름:

1. 목표 문장과 인식 문장을 normalize
2. `levenshtein` 거리 계산
3. 최대 길이 기준으로 유사도 퍼센트 계산

개념상:

`정확도 = (1 - 편집거리 / 최대문자길이) * 100`

즉 현재는 음운론/억양보다 **문자열 유사도 기반 정확도**가 핵심이다.

---

## 7. 전투 구조

문장 마법은 `HP 전투형` 구조다.

기본값:

- playerHp = 3
- enemyHp = 3

라운드 흐름:

1. 현재 문제 표시
2. 사용자 녹음
3. STT 전사
4. 정확도 비교
5. 기준 이상이면 적 HP 감소
6. 기준 미만이면 내 HP 감소
7. 중간 결과 모달 표시
8. 다음 문제로 진행 또는 최종 승패 판정

즉 한 문제 = 한 번의 공격/피격 판정이다.

---

## 8. 라운드/상태 표시

상단에는 `BattleStatusBar`가 있다.

표시 요소:

- 플레이어 아이콘
- 플레이어 HP
- ROUND
- 적 HP
- 적 아이콘

최근 사용자 요청으로:

- 상태바 크기/아이콘 크기 조정
- 녹음 중 오라
- 분석 중 적 피격 효과
- 중앙 진행 바 반응

등을 추가해 전투 감각을 강화했다.

## 9. 모드별 본문 표시

### 문장 만들기

- 정답 문장 조각을 섞어서 칩 형태로 표시
- 사용자는 올바른 문장을 말함

### 잰말 놀이

- 목표 문장을 그대로 크게 표시
- 사용자는 문장을 또렷하게 읽음

즉 두 모드 모두 최종 판정은 transcript vs targetSentence 비교지만,  
사용자에게 보여주는 본문은 다르다.

## 10. 결과 처리

결과 모달은 [LingoResultModalShell.tsx](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/components/lingo/LingoResultModalShell.tsx) 기반으로 정리되어 있다.

### 중간 전투 결과

- 공격 성공 / 피격
- 정확도
- 인식 문장 요약
- 다음 문제로 이동

중간 결과는 최근 사용자 요구에 따라 **스크롤 없는 컴팩트 모달**로 축소되었다.

### 최종 승패 결과

- `VICTORY` 또는 `DEFEAT`
- 전투 연출 카드
- 최종 정확도/인식 결과
- 단계 선택 이동
- 마지막 결과일 때만 홈 버튼 표시

---

## 11. 점수/평가 기준

문장 마법에서 실질적으로 중요한 평가지표는 다음과 같다.

- 정확도(`accuracy`)
- 내 HP / 적 HP
- 승패 여부

즉 일반적인 누적 점수보다 **정확도에 따른 전투 결과**가 더 중요한 구조다.

---

## 12. 관리자/디버그 요소

사용자 요청에 따라:

- 본문에 있던 `성공 예시`, `실패 예시`는 제거
- 관리자/로컬 환경일 때만 상단 헤더에 표시

즉 일반 사용자 화면은 훈련 흐름만 보이고, 예시 버튼은 디버그용으로 분리됐다.

---

## 13. 현재 UI 구성

현재 메인 화면은 다음 구조다.

1. 상단 배틀 상태바
2. 메인 주문 카드
   - 모드 배지
   - 목표 문장 또는 문장 조각
   - 녹음 버튼
   - 실시간 인식 결과 영역
3. 분석 중 모달
4. 중간 결과 모달
5. 최종 승패 모달

즉 좌우 패널 없이 `1열 중심 전투 화면`이다.

---

## 14. 장점

- 문장 단위 발화를 게임화하기 좋음
- 정확도 판정을 HP 전투와 연결해 동기 부여가 큼
- 문장 만들기와 잰말 놀이를 한 게임 안에 묶어 학습 폭이 넓음
- 서버 STT 기반이라 브라우저별 편차가 상대적으로 적음

---

## 15. 현재 구현상 주의점

- 정확도는 현재 문자열 유사도 기반이므로, 억양/속도/발성 품질 자체를 직접 평가하진 않는다.
- 서버 STT 응답 지연이 길면 전투 템포가 느려질 수 있다.
- 긴 문장은 transcript 품질에 따라 체감이 크게 달라진다.
- 승패 모달은 공통 셸을 쓰지만, 문장 마법은 전투 전용 정보가 많아 내용 길이 관리가 중요하다.

---

## 16. 향후 개선 제안

- 정확도 외에 발화 속도/끊김/유창성 보조 점수 추가
- 잰말 놀이에서 threshold 외에 시간 제한 요소 도입
- 문장 만들기에서 단어 배열 시각 힌트 강화
- 결과 모달에서 발화 피드백 문구를 더 세분화

---

## 17. 한 줄 요약

문장 마법은 **문장 만들기와 잰말 놀이를 통해 발화 정확도를 측정하고, 그 결과를 전투 HP 시스템으로 표현하는 서버 STT 기반 문장 훈련 게임**이다.
