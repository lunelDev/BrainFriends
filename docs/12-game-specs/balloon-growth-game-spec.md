# 풍선 키우기 게임 기획/구현 문서

## 1. 개요

`풍선 키우기`는 음성의 **크기와 유지력**을 기반으로 풍선을 목표 크기까지 성장시키는 훈련형 게임이다.  
사용자는 제한 시간 안에 적절한 크기의 발성을 유지해야 하며, 지나치게 큰 발성을 오래 유지하면 실패한다.

현재 구현은 **텍스트 STT가 아닌 음량 기반 입력 게임**이다.  
즉, 무엇을 말했는지가 아니라 **얼마나 안정적으로 소리를 냈는지**가 핵심이다.

---

## 2. 게임 목표

- 일정 시간 안에 풍선을 목표 크기까지 키운다.
- 너무 작지도, 너무 크지도 않은 발성을 유지한다.
- 위험 구간 발성을 오래 유지하면 풍선이 터진다.

사용자에게 중요한 피드백은 다음 4가지다.

- 현재 음량
- 남은 시간
- 위험 누적
- 목표 도달 진행률

---

## 3. 난이도 구성

난이도 데이터는 [balloonGrowthData.ts](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/data/balloonGrowthData.ts)에 정의되어 있다.

### 하

- 제한 시간: `15초`
- 시작 크기: `18`
- 안전 구간: `35 ~ 70`
- 위험 시작: `85 이상`
- 위험 유지 허용 시간: `400ms`
- 기본 성장 속도: `18 / sec`

### 중

- 제한 시간: `18초`
- 시작 크기: `16`
- 안전 구간: `38 ~ 68`
- 위험 시작: `84 이상`
- 위험 유지 허용 시간: `350ms`
- 기본 성장 속도: `15 / sec`

### 상

- 제한 시간: `20초`
- 시작 크기: `14`
- 안전 구간: `40 ~ 65`
- 위험 시작: `82 이상`
- 위험 유지 허용 시간: `300ms`
- 기본 성장 속도: `13 / sec`

공통값:

- 목표 크기: `100`
- silenceThreshold는 남아 있지만, 현재는 자동 감소를 거의 쓰지 않기 때문에 직접적인 실패 조건은 아니다.

## 4. 입력 방식

풍선 키우기는 [useBalloonAudioInput.ts](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/lib/audio/useBalloonAudioInput.ts)를 사용한다.

구성 목적:

- 일반적인 `useAudioAnalyzer`보다 더 민감하게 발성을 감지
- `아~` 같은 지속 발성에 즉시 반응
- 텍스트 인식 없이 음량만 게임에 전달

현재 표시되는 숫자는 실제 물리 dB라기보다 **게임용 정규화 입력 강도**에 가깝다.

## 5. 라운드 구조

게임 흐름은 [BalloonGrowthGame.tsx](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/components/lingo/BalloonGrowthGame.tsx) 기준으로 다음과 같다.

1. 사용자가 난이도를 선택한다.
2. 마이크 입력을 시작한다.
3. 게임 phase가 `playing`으로 전환된다.
4. 100ms 단위 루프에서 풍선 크기, 위험 누적, 남은 시간을 갱신한다.
5. 목표 크기에 도달하면 성공한다.
6. 위험 구간을 오래 유지하면 실패한다.
7. 시간이 끝나면 실패한다.

Phase:

- `select`
- `playing`
- `success`
- `fail`

## 6. 성장 판정 방식

현재 구현은 사용자의 최근 요구를 반영해 **발성이 들어오기만 해도 성장 체감이 나도록 완화**된 상태다.

### 6-1. 안전 구간

입력이 안전 구간 안에 있으면 가장 안정적으로 성장한다.

- 안전 구간 중심에 가까울수록 성장 효율 증가
- 최적 구간이면 메시지도 긍정적으로 바뀜

### 6-2. 안전 구간 바로 아래

안전 구간보다 조금 낮아도, 발성이 일정하게 유지되면 느리게 성장한다.

이 로직은 “정확히 safeMin 이상만 성장”보다 덜 답답하게 만들기 위한 장치다.

### 6-3. 일반 발성 구간

현재는 위험 구간만 아니면 **실제 발성이 들어오고 있다는 감각**을 살리기 위해, 일반 발성 구간에서도 성장하도록 조정돼 있다.

즉 사용자 체감은:

- 소리를 내면 커짐
- 더 적절한 구간일수록 더 잘 커짐

에 가깝다.

### 6-4. 자동 감소

현재는 사용자 요청에 따라 **자동 감소가 없다.**

즉:

- 잠깐 쉬어도 바로 줄지 않음
- 오래 멈춰도 자동으로 다시 작아지지 않음

---

## 7. 위험 판정

위험 구간은 `effectiveVolume >= dangerMin`일 때 시작한다.

이때:

- `dangerMsRef`가 누적됨
- `dangerRatio`가 0~100으로 표시됨
- 누적 시간이 `dangerHoldMs`를 넘으면 `burst` 실패 처리

즉 실패 기준은 “너무 큰 소리를 한 번 냈다”가 아니라  
“위험한 크기를 일정 시간 이상 유지했다”이다.

---

## 8. 시간 처리

남은 시간은 `timeLeft`로 관리되며, 게임 루프마다 감소한다.

- `timeLeft <= 0`이 되면 `timeout` 실패
- 성공은 시간 안에 목표 크기 도달

UI에는 상단 `Time / Progress / Level` 카드로 표시된다.

---

## 9. 결과 계산

결과는 `buildResult(type)`에서 만든다.

### 성공

- title: `풍선 완성!`
- subtitle: `안전한 소리를 잘 유지해서 목표 크기에 도달했어요.`

### 실패

- 터짐: `풍선이 터졌어요`
- 시간 초과: `시간 초과`

### 결과 지표

- 발성 시간: `voicedMs / 1000`
- 평균 음량: `totalVolume / sampleCount`
- 안정성: `safeMs / voicedMs * 100`

즉 풍선 키우기 점수는 전통적 score보다 **결과 리포트용 성과 지표**에 가깝다.

### 성공

- phase를 `success`로 변경

## 12. 관리자/디버그 요소

현재 게임에는 `테스트 발성 올리기` 버튼이 있다.

이 버튼은:

- 마우스로 누르고 있는 동안
- 실제 입력이 약해도
- 안전 구간에 가까운 가상 입력을 만들어 테스트 가능

즉 실제 사용자의 훈련용 기능이라기보다는 **튜닝/디버깅용 성격**이 강하다.

---

## 13. 장점

- 텍스트 인식 없이도 즉시성 있는 음성 훈련 가능
- 사용자가 소리를 내면 바로 반응하는 구조로 체감이 좋음
- 위험 구간과 안정 구간을 분리해 훈련 방향이 명확함
- 짧은 세션으로 반복하기 좋음

---

## 14. 현재 구현상 주의점

- 숫자 `dB` 표기는 실제 측정 dB가 아니라 게임용 환산값이다.
- 안전 구간이 완화되어 있어, 현재는 “소리를 내면 커진다” 체감이 우선이다.
- 자동 감소가 없으므로, 난이도는 사실상 `시간 제한 + 위험 누적` 중심이다.
- 너무 민감하거나 덜 민감한 문제는 마이크 환경에 따라 달라질 수 있다.

---

## 15. 향후 개선 제안

- 입력 강도 라벨을 `dB` 대신 `입력 강도`로 변경
- 성장 구간/위험 구간을 미터에서 더 명확히 시각화
- 난이도별 목표 크기나 목표 패턴을 다르게 구성
- 긴 호흡 유지 미션, 구간별 목표치 미션 등 확장

---

## 16. 한 줄 요약

풍선 키우기는 **적절한 크기의 발성을 일정 시간 유지해 풍선을 목표 크기까지 키우는 음량 기반 훈련 게임**이며, 현재 구현은 “소리를 내면 바로 성장 체감이 나는 것”을 최우선으로 조정돼 있다.
