# 게임 모드 통합 기획서

## 1. 문서 목적

이 문서는 `브레인프렌즈 게임 모드` 전체를 한 번에 이해할 수 있도록 정리한 통합 기획서다.  
현재 프로젝트에 구현된 게임은 다음 4종이다.

- `한글 테트리스`
- `말로 열기`
- `문장 마법`
- `풍선 키우기`

각 게임은 서로 다른 훈련 목표를 갖지만, 공통적으로 **음성 기반 상호작용**을 핵심으로 한다.

---

## 2. 게임 모드 전체 목표

게임 모드의 목적은 단순 오락이 아니라, 사용자가 반복적으로 음성/언어 훈련을 하도록 만드는 것이다.

공통 목표는 다음과 같다.

- 사용자가 지루하지 않게 반복 발화 훈련을 하도록 유도
- 언어 요소를 게임 규칙과 연결해 몰입도 향상
- 결과를 점수/성공/실패/성장 등으로 즉시 피드백
- 훈련 세션의 난이도를 조절해 단계별 접근 가능

즉 게임 모드는 `치료/훈련 목표 + 반복 동기 + 실시간 피드백`을 결합한 층위다.

---

## 3. 공통 UX 원칙

현재 게임 모드 화면 설계는 다음 원칙을 따른다.

### 3-1. 사용자 화면은 단순하게

사용자에게 보여주는 핵심은 다음으로 제한한다.

- 현재 해야 할 행동
- 현재 목표
- 진행 상태
- 방금 결과

반대로 기술적 상세 정보는 최대한 숨긴다.

### 3-2. 디버그 정보는 관리자 전용

- 정답 강제 처리
- 실패 강제 처리
- 예시 버튼
- 음성 인식 상세 상태

같은 요소는 일반 사용자 화면이 아니라 관리자/로컬 모드에서만 노출한다.

### 3-3. 결과 모달은 공통 톤 사용

현재 결과 모달은 [LingoResultModalShell.tsx](c:/Users/pc/Desktop/ProjectFiles/BrainFriends/src/components/lingo/LingoResultModalShell.tsx) 기반으로 묶여 있다.

즉 게임마다 내용은 달라도,

- 전체 오버레이
- 카드 외곽
- CTA 버튼
- 결과 요약 카드

는 같은 계열로 유지한다.

## 4. 게임별 역할 요약

### 한글 테트리스

- 목표: 단어/구/문장을 정확하게 발화
- 핵심 재미: 낙하 압박 + 음성 판정
- 성격: 반응 속도와 정확도를 함께 요구하는 실시간 게임

### 말로 열기

- 목표: 그림을 보고 카테고리를 말하기
- 핵심 재미: 제한 시간 안에 빠르게 분류
- 성격: 카테고리 개념화 + 짧은 음성 반응

### 문장 마법

- 목표: 문장을 정확히 말해 전투에서 승리
- 핵심 재미: 발화 정확도를 HP 전투로 표현
- 성격: 문장 단위 발화 평가형 게임

### 풍선 키우기

- 목표: 적절한 발성을 유지해 풍선을 목표 크기까지 성장
- 핵심 재미: 소리 크기와 유지력의 즉시 반응
- 성격: 음량/호흡 조절형 게임

## 5. 입력 방식 비교

게임마다 입력 방식이 다르다. 이 차이를 명확히 구분하는 것이 중요하다.

### 5-1. 한글 테트리스

- 현재 방향: 브라우저 음성 인식 중심
- 이유: 짧은 단어/구에서 즉시 반응성이 중요
- 주의: 긴 문장 판정과의 균형 필요

### 5-2. 말로 열기

- 브라우저 SpeechRecognition 사용
- 이유: `과일 / 동물 / 탈것` 같은 짧은 분류어를 빠르게 인식해야 함
- 주의: Chrome 계열 브라우저 의존성이 큼

### 5-3. 문장 마법

- 서버 STT(`/api/proxy/stt`) 사용
- 이유: 문장 전체 transcript를 받아 정확도 비교 필요
- 주의: 네트워크/서버 응답 속도에 영향 받음

### 5-4. 풍선 키우기

- 텍스트 STT 없음
- 음량 기반 오디오 입력만 사용
- 이유: 무엇을 말했는지가 아니라 소리 크기와 유지가 중요

즉 게임 모드는 하나의 입력 방식으로 통일된 시스템이 아니라,  
**게임 목적에 따라 음성 처리 방식을 다르게 채택하는 구조**다.

---

## 6. 난이도 체계 비교

### 한글 테트리스

- 레벨 1~9
- 단어에서 긴 문장까지 확장
- 난이도에 따라 목표 길이와 판정 부담 증가

### 말로 열기

- 쉬움 / 보통 / 어려움
- 카테고리별 카드 수 증가
- 카드당 제한 시간 감소

### 문장 마법

- 문장 만들기 / 잰말 놀이 모드
- 잰말 놀이: 하/중/상 threshold 증가
- 높은 난이도일수록 더 어려운 문장 + 더 높은 정확도 요구

### 풍선 키우기

- 하 / 중 / 상
- 제한 시간, 안전 구간, 위험 구간, 성장 속도 차이

즉 모든 게임은 “난이도 상승”을 갖고 있지만, 그 방식은 게임별로 다르다.

## 7. 점수/성과 측정 방식 비교

### 한글 테트리스

- 음성 정확도
- 성공/실패 누적
- 문제 흐름
- 최종 결과 리포트

### 말로 열기

- 열린 카드 수
- 정확도
- 진행률
- 걸린 시간

### 문장 마법

- 문장 정확도
- 플레이어 HP / 적 HP
- 승패 결과

### 풍선 키우기

- 발성 시간
- 평균 음량
- 안정성
- 목표 도달 여부

즉 게임마다 “점수”의 정의가 다르며, 공통 점수 체계 하나로 묶여 있지 않다.

## 8. 게임별 현재 구조

### 8-1. 한글 테트리스

기본 구조:

- 상단 목표 카드
- 중앙 보드
- 우측 상태판

핵심 요소:

- 현재 목표 발화
- 낙하 블록
- 문제 흐름
- 음성 입력 상태
- 카메라/모니터링

현재 방향:

- 긴 문장 처리 로직은 단순 단어 매칭보다 더 정교한 분절 전략이 필요함

### 8-2. 말로 열기

기본 구조:

- 메인 카드
  - 카테고리 안내
  - 그림
  - 메인 질문
  - 자동 듣기 상태
- 우측 보조 카드
  - Listening Stream
  - Session Progress
- 우측 모니터링 패널

핵심 요소:

- 제한 시간 안에 분류어 즉시 판정
- 카드 전환 시 carry-over 방지
- 자동 듣기 유지

### 8-3. 문장 마법

기본 구조:

- 상단 배틀 상태바
- 중앙 주문 카드
- 녹음 버튼
- 실시간 인식 영역
- 전투/최종 결과 모달

핵심 요소:

- 문장 transcript vs 목표 문장 정확도 비교
- 전투 HP 구조
- 문장 만들기 / 잰말 놀이 통합

### 8-4. 풍선 키우기

기본 구조:

- 상단 Time / Progress / Level
- 중앙 풍선 스테이지
- 하단 Live Volume / Danger Accumulation
- 결과/선택 모달

핵심 요소:

- 소리를 내면 성장
- 위험 구간 누적 시 실패
- 자동 감소 없음

---

## 9. 공통 모달/선택 화면 구조

게임 모드 전반에서 통일한 항목은 다음과 같다.

### 결과 모달

- 공통 결과 셸 사용
- 게임별 수치 카드만 다르게 표시
- 최종 CTA는 게임/상황에 따라 분기

### 선택 모달

- 카드형 전체 화면 모달
- 상단 아이콘 + 제목 + 설명
- 난이도 카드 선택
- 하단 시작 버튼

### 관리자용 헤더 액션

- 게임에 따라 상단 헤더에 디버그 버튼 제공
- 예: 정답 처리, 실패 처리, 예시 보기

---

## 10. 관리자/운영 기준

현재 운영 기준은 다음과 같다.

- 일반 사용자
  - 게임 진행에 필요한 정보만 노출
- 관리자/로컬 모드
  - 디버그 버튼, 예시 버튼, 강제 판정 버튼 노출 가능

또한 최근 정책상:

- `자가진단`, `언어재활`의 서버 로직은 건드리지 않음
- 게임 모드 쪽에서만 음성/서버 로직 조정

으로 운영 중이다.

## 11. 현재 구현상 리스크

### 11-1. 브라우저 음성 인식 의존성

- 말로 열기, 테트리스 일부는 브라우저 음성 인식 품질 영향을 받음
- Chrome 계열 브라우저를 기준으로 보는 것이 현실적

### 11-2. 긴 문장 vs 즉시 판정

- 테트리스는 짧은 단어와 긴 문장이 섞여 있어, 한 가지 판정 전략으로 해결하기 어려움

### 11-3. 실시간 체감

- 사용자는 “입력 들어옴 → 판정됨”이 바로 느껴져야 함
- 기술적으로는 되더라도 UX가 약하면 실패처럼 느껴질 수 있음

### 11-4. 서버 STT 의존

- 문장 마법은 API 키, 네트워크, upstream 상태에 영향 받음

## 12. 향후 발전 방향

### 공통

- 게임별 입력 방식과 지원 브라우저를 명확히 문서화
- 디버그/관리자 요소를 더 분리
- 결과 리포트의 표현 방식 통일

### 한글 테트리스

- 긴 문장 분절 평가 고도화
- 음성 판정 안정성 개선

### 말로 열기

- 브라우저 인식 지원 여부 안내
- 자동 듣기 상태 피드백 강화

### 문장 마법

- 정확도 외 발화 품질 보조 지표 추가
- 전투 모달 길이 최적화

### 풍선 키우기

- 성장 구간 시각화 강화
- 난이도별 목표 패턴 다양화

## 14. 한 줄 요약

브레인프렌즈 게임 모드는 **음성 입력을 중심으로 서로 다른 언어 훈련 목표를 게임화한 4종의 훈련 시스템**이며, 각 게임은 목적에 맞게 입력 방식·난이도·판정 구조를 달리 가져가되 결과 경험은 공통 디자인 시스템으로 묶는 방향으로 설계되어 있다.
