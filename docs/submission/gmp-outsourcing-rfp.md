## 2. 프로젝트 개요

### 2.1 제품 개요
- **제품명**: 브레인프렌즈 (BrainFriends)
- **제품 성격**: 실어증(aphasia) 및 언어장애 환자용 언어재활 디지털치료기기 (DTx, Digital Therapeutic)
- **플랫폼**: 웹/모바일 기반 소프트웨어 의료기기 (SaMD, Software as a Medical Device)
- **주요 기능**:
  - 자가진단 모드 (K-WAB 기반 언어평가)
  - 언어재활 훈련 모드 (음향·조음·얼굴 표정 멀티모달 피드백)
  - 노래 기반 재활 모드 (멜로디 억양 치료, MIT 기반 응용)
- **핵심 기술 스택**:
  - Frontend: Next.js
  - Backend: FastAPI + Parselmouth (음향 분석) + OpenAI Whisper (음성인식) + MediaPipe (얼굴 랜드마크)
  - 결정론적 음향 파라미터 추출 (REQ-ACOUSTIC-001~012 구현 완료)

### 2.2 등급 예상
- **예상 등급**: 2등급 의료기기 (유력)
- **근거**: 언어재활 치료 목적, 진단 보조 아님, 환자 직접 치료 개입
- **확정 경로**: KTL 사전 문의 및 식약처 분류 확인 절차 진행 중

### 2.3 사용 목적 (Intended Use) 요약
- 의료진의 처방 하에 실어증 환자의 언어 기능 회복을 보조하는 소프트웨어 치료기기
- 병원 내 SLP(언어재활사) 감독 하 또는 재택 사용
- 사용 환경: 만 18세 이상 실어증 진단 환자, 의사 처방 기반

### 2.4 현재 개발 단계

- 사이버보안 정책 초안, AI 성능평가 구조 초안 보유

## 3. 요청 범위 (상세)

본 컨설팅 계약의 범위는 아래 A~G 항목 전부를 포함한다.
각 항목별 수행 방식·산출물·검수 기준을 제안서에 명시할 것.

### A. 품질매뉴얼 (Quality Manual) 작성 지원
- ISO 13485:2016 기반 품질매뉴얼 초안 작성
- 당사 조직구조·프로세스에 맞춘 커스터마이즈
- 최고경영자 선언문, 품질방침, 품질목표 포함
- 문서화 체계 (Document Control) 정립

### B. 표준작업절차 (SOP) 5종 초안 작성
1. **설계관리 SOP** (Design Control) — ISO 13485 7.3 대응
2. **유지보수 SOP** (Servicing) — 소프트웨어 업데이트 포함
3. **문제해결 SOP** (CAPA, Corrective and Preventive Action)
4. **형상관리 SOP** (Configuration Management) — 소프트웨어 버전 관리 연계
5. **보안지침 SOP** (Cybersecurity / Information Security) — 식약처 사이버보안 가이드라인 대응

### C. ISO 14971:2019 위험관리 파일 구축
- Risk Management Plan 작성
- Hazard Identification (위험 식별) 워크숍 진행
- Risk Analysis / Evaluation (위험 평가) — Severity × Probability Matrix
- Risk Control Measures 적용 (기술·문서·교육적 통제)
- Residual Risk Evaluation
- Risk Management Report 작성

### D. IEC 62304:2006+A1:2015 소프트웨어 안전 등급 분류 및 추적성
- Software Safety Classification (Class A/B/C 판정)
- 브레인프렌즈 SW 아키텍처 문서화 지원
- SW 요구사항 ↔ 설계 ↔ 구현 ↔ 테스트 추적성 매트릭스
- SOUP (Software of Unknown Provenance) 관리 — Parselmouth, Whisper, MediaPipe 등 오픈소스 라이브러리
- SW 유지보수 프로세스 정립

### E. IEC 62366-1:2015 사용적합성 평가 지원
- Use Specification 문서화
- Use-related Risk Analysis
- Formative Evaluation (포메이티브 평가) 설계·실행 지원
- Summative Evaluation (서머티브 평가) 프로토콜·실행 지원
- Usability Engineering File 구성

### F. GMP 적합판정 심사 대응
- 심사 기관 선정 자문 (식약처, KTR, KTL 등)
- 사전 점검 (Mock Audit) 1회 이상 진행
- 현장 심사 동행 (대면)
- 부적합 사항(Non-conformity) 대응 및 후속조치 관리
- 최종 GMP 적합판정서 취득까지 전 과정 지원

### G. 허가 후 변경관리·사후관리 교육
- Change Control 프로세스 교육
- Post-Market Surveillance (PMS) 체계 구축 자문
- Adverse Event 리포팅 절차 교육
- 정기 품질 심사 내부 심사원 양성 교육 (당사 인력 2~3명)

1. **1차 접촉**: 공식 홈페이지 문의 폼 또는 대표 이메일
   - 요지: "SaMD/DTx 품목허가 대상 QMS 구축 컨설팅 RFP 발송 예정, 수령 의향 확인"

   - 접촉 후 담당자 이메일 확인 → 개별 메일 발송

4. **질의 핸들링 원칙**
   - 한 업체가 한 질의 → 다른 업체에도 공유 (Q&A 정리 회람)
   - 민감 정보(타 업체 견적·전략)는 절대 누설 금지

5. **인터뷰 진행**
   - 60~90분 (업체 발표 30분 + Q&A 30분 + 팀 대화 30분)
   - 참석자: 대표 이현송, 팀장 박지수, (가능시) 외부 자문 RA 1인

### 주의사항 (법적·윤리적)

- RFP는 **계약 청약이 아님**. 최종 계약은 별도 협상·체결.
- 업체로부터 받은 제안 내용은 **제3자 공유 금지**.
- 당사의 기술·전략 정보는 **NDA 체결 후** 상세 공개.
- 공공 R&D 과제 연계 시 업체 선정은 해당 과제 규정을 우선 준수.
