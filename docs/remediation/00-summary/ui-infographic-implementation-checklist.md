# UI 인포그래픽 구현 체크리스트

## 목적

이 문서는 사용자/치료사 UI 인포그래픽 기준으로 현재 프로젝트에 이미 반영된 항목과 아직 반영되지 않은 항목을 구분해서 보기 위한 체크리스트입니다.

## 1. 치료사 화면 인포그래픽 기준

### 이미 반영된 항목

- 치료사 전용 라우트 분리
  - `src/app/therapist/*`
- 치료사 대시보드 진입점 존재
- 치료사 대시보드에 핵심 KPI, 빠른 검색, 운영자 액션 영역 반영
- 치료사 대시보드에 최신 사용자 기준 상세 핵심 패널(AQ 추이, Step1~6, 측정 품질) 반영
- 사용자 목록 화면 존재
- 사용자 목록 화면의 검색/바로가기 흐름 정리
- 결과 화면에 검색/필터 UI 반영
- 대시보드와 결과 화면에서 V&V/AI export, 저장 실패 바로가기 등 즉시 실행 액션 반영
- 사용자 상세 화면 존재
- 사용자 상세 화면에서 결과 JSON 다운로드 및 저장 재시도 액션 반영
- AQ, Step 결과, 측정 품질, 메모, follow-up 확인 가능
- V&V / 보안 / AI 평가 운영 화면 존재
- V&V export 가능
- AI 평가 export 가능

### 아직 부족한 항목

- 운영자 액션 버튼(리포트 다운로드, 과제 배정 등)의 실제 기능 확대
- 인포그래픽 수준의 카드 밀도와 시각 계층 추가 고도화
- 결과 화면과 메인 대시보드 사이의 상태 연계(선택 사용자 유지, 필터 연동) 추가 고도화

### 현재 판단

치료사 UI는 **핵심 KPI, 빠른 검색, 사용자 상세 핵심 패널, 결과 필터, 즉시 실행 액션, 일부 다운로드/재시도 기능까지는 반영됐으나, 액션 범위 확대와 화면 간 연동 고도화는 아직 부족**합니다.

## 2. 사용자 화면 인포그래픽 기준

### 이미 반영된 항목

- 홈/모드 선택 화면을 대시보드형으로 일부 정리
- 최근 AQ, 목표, 측정 품질, 최근 결과 등 일부 홈 정보 노출
- 결과 리포트에 성장 확인형 상단 요약 일부 반영
- 결과 페이지의 server-first 동기화 구조 반영
- `마이페이지 / 내 재활 관리` 기본 화면 추가

### 아직 부족한 항목

- 인포그래픽 수준의 홈 화면 카드 밀도와 마이페이지 완성도 고도화
- 사용자 훈련 리스트 화면의 추천형/진행형 정리
- 결과 리포트의 시각적 완성도와 사용자 동기부여형 표현
- 마이페이지/내 재활 관리 화면의 상세 기능 보강

### 현재 판단

사용자 UI는 **일부 구조만 반영된 상태이며, 인포그래픽 수준의 완성형 UX는 아직 아님**으로 판단합니다.

## 3. Step 훈련 진행 화면

### 현재 상태

- step 화면은 기존 집중형 인터랙션 디자인 유지
- 과도한 대시보드형 상단 패널은 제거

### 판단

step 화면은 현재 인포그래픽보다 **기존 집중형 UX를 유지하는 방향으로 확정**된 상태입니다.

## 4. 전체 판단

### 이미 완료된 축

- 공인성적서 대응 구조
  - SW V&V
  - 사이버보안
  - AI 성능평가

### 부분 반영된 축

- 치료사 UI
- 사용자 홈/결과 UX

### 아직 본격 작업 전인 축

- 인포그래픽 수준의 사용자/치료사 UI 완성도 정리

## 5. 다음 구현 우선순위

1. 치료사 메인 대시보드 통합형 레이아웃 정리
2. 사용자 홈 / 결과 리포트 톤 정리
3. 검색/필터 / 운영자 액션 통합
4. 마이페이지 / 내 재활 관리 화면 추가 검토

## 6. 한 줄 정리

현재 프로젝트는 **인포그래픽 기준 UI 완성보다 공인성적서 대응 구조 구현이 더 앞서 있으며, UI는 일부 반영 상태**입니다.
