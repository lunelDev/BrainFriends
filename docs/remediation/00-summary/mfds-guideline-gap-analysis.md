# 0. 총평 (한 페이지 요약)

## 0-1. 가장 적용성 높은 가이드라인 Top 3
1. **PDF #6 디지털의료기기소프트웨어 허가·심사 가이드라인** (안내서-1425-01, 2025.05) — **직접 적용**. 브레인프렌즈는 "독립형 디지털의료기기소프트웨어(SaMD)"의 정의(2조 1호)에 정확히 부합. 신청서 13개 기재 항목(제9~20조)이 곧 허가 제출 구조.
2. **PDF #7 디지털치료기기 허가·심사 가이드라인** (안내서-1045-02, 2025.05) — **직접 적용 가능성 높음**. "의학적 장애나 질병을 예방·관리·치료하기 위해 환자에게 근거 기반의 치료적 개입을 제공하는 SaMD" 정의(§I.3.가)에 실어증 재활훈련 기능이 부합. 단 "치료 작용기전의 과학적(임상적) 근거"(§II.3) 요건과 전향적 임상시험(§III.2.다) 필수 여부가 결정적 분기점.
3. **PDF #5 디지털의료기기 GMP 가이드라인** (안내서-1457-01, 2025.11) — **허가 전 의무**. 「디지털의료제품법」 제24조에 따라 제조·수입하려면 **적합판정(GMP 심사)을 반드시 선행**해야 하며, 유효기간 3년·정기심사(§시행규칙 제35조). 독립형 SaMD는 [별표3] 소프트웨어 심사기준(p.106~225) 적용.

## 0-2. 제품 규제 분류 추정
- **1차 분류: 디지털의료기기소프트웨어(SaMD)** — PDF #6 §II.2 정의에 완전 부합. 독립형.
- **2차 분류: 디지털치료기기(DTx) 가능성 높음** — PDF #7 §I.3.가 "재활 보조가 아닌 치료적 개입"에 해당. 단 판단은 ①국제질병분류(실어증=R47.0, 디스아트리아=R47.1 등 명확) ②전문의 진단에 기반한 환자 대상성 ③치료 작용기전의 과학적 근거 3요건 동시 충족 필요 (§II 판단 흐름도).
- **3차 분류: 인공지능 기술 적용 여부** — Whisper STT·LLM 피드백을 "기계학습 기반 디지털의료기기(MLMD)"로 볼지 여부. PDF #2 §II.2 "기계학습 적용 SW에 대한 규제적 접근". 만약 AI 모델이 핵심 성능의 일부라면 PDF #2 전체(훈련데이터 관리·성능·편향 분석 등) 추가 적용.
- **등급 추정**: PDF #3 보도참고 <붙임> 등급별 사례 표 기준 → **2등급**이 가장 유력. 근거: "마약류 중독에 따른 정신·행동장애에 대한 동기강화 및 인지행동치료 요법 제공"이 3등급 예시에 있으나 침습성·긴급성이 낮은 재활훈련은 2등급 예시("근감소증에 근력강화운동 제공형")에 더 가까움. 단 치료적 개입의 심각도 판단은 최종적으로 식약처가 결정.

## 0-3. 가장 먼저 확보해야 할 문서·증적 Top 5
1. **SaMD 안전성 등급 판단서** (A/B/C) — PDF #1 §IV.3, PDF #6 §IV. 위해요인 표 + 심각도 산정 + 등급결정 로직. 이것이 없으면 IEC 62304 적용 범위가 결정되지 않음.
2. **ISO 14971 위험관리 파일** — PDF #5 [별표3] 2.1.1~2.1.6, PDF #2 §III.2, PDF #7 §III 전반. 위험분석·평가·통제·잔여위험·위험관리보고서.
3. **SRS / SDS / V&V 추적성 매트릭스** — PDF #5 [별표3] 1.1.1(개발계획)·1.2(요구사항 분석)·1.3(설계)·1.4(검증). 이미 일부 구현됨(`src/lib/vnv/*`) → 표준 문서 포맷으로 치환.
4. **품질매뉴얼·품질방침·SOP 세트** — PDF #5 [별첨1~5] 샘플 5종(설계및개발 절차서, 유지보수 절차서, 문제해결 절차서, 형상관리 절차서, 보안지침 절차서). **제조업허가 전 GMP 심사에 필수**.
5. **치료 작용기전의 과학적(임상적) 근거 자료** — PDF #7 §II.3 / §III.2.가. 대한의학회 CPG, 동료심사 논문, 연구자 임상시험 자료 중 하나. SaMD 정의만으로는 DTx 인정 안 됨.

## 1-1. 의료기기 소프트웨어 허가·심사 가이드라인 (PDF #1, 안내서-0612-05, 2025.05)

**범위 요약**: 2025년 5월 개정으로 **"내장형 소프트웨어"에 한정**되도록 적용범위가 좁혀짐 (§I.2). 초음파영상진단장치 등 하드웨어에 내장된 SW 및 내장형 SW와 연동하는 앱/웹을 다룸. 독립형은 PDF #6을 참조하도록 명시.

**적용성: 부분 적용 (개념 참조용)**
- 근거: 브레인프렌즈는 독립형 SaMD라 직접 대상이 아님. 단 §IV "SW 안전성 등급(A/B/C)" 프레임워크는 PDF #6도 그대로 인용(§IV "'의료기기 소프트웨어 허가·심사 가이드라인' Ⅳ 참조")하므로 **개념·판단 절차는 유효**.

**핵심 요구사항** (브레인프렌즈에 적용되는 것만)
- §IV.1 개발·위험관리: ISO 14971 프로세스(위험분석→평가→통제→잔여위험 허용성→보고서→생산후 정보) 필수.
- §IV.2~3 안전성 등급 결정 절차: <그림5> 등급판단 플로우차트(심각도×발생가능성 100%).
- §V 기술문서 작성 공통사항 — **독립형엔 직접 인용 안 되지만** 위해요인 표 형식(<표3>)은 SaMD에도 동일 활용 가능.

**현재 상태 vs 요구**
| 항목 | 상태 |
|---|---|
| ISO 14971 위험관리 프로세스 | ✗ 미충족 (파일 자체 없음) |
| SW 안전성 등급 판단 근거 문서 | ✗ 미충족 |
| 위해요인 표(주요기능×Hazard×Harm×Severity×Risk) | ✗ 미충족 |

**필요 조치**
- **T-REG-001** SW 안전성 등급 판단서 작성: 실어증 재활훈련 중 잘못된 피드백이 초래할 위해 → "심각한 상해 또는 사망 가능성 없음(A)" 혹은 "경상 가능성(B)" 중 하나로 귀결될 가능성이 높음. 심각도 1~5 척도 설정 후 판단 근거 서면화. (담당: E 규제·특허팀)

## 1-2. 인공지능기술이 적용된 디지털의료기기 허가·심사 가이드라인 (PDF #2, 안내서-0804-04, 2025.05)

**범위 요약**: "기계학습 가능 의료기기(MLMD)" 정의, 훈련데이터 정보·업데이트 주기·클라우드 서비스 기재(§III.1), 성능(민감도/특이도/AUC) 검증(§III.2), 전향적/후향적 임상시험 설계(§III.3), 훈련데이터셋 관리정책 필수(§III.6).

**적용성: 조건부 적용 — 제품 내 AI의 역할 규정에 따라 갈림**
- 근거: 브레인프렌즈는 Whisper STT(OpenAI 상용)와 Parselmouth 수학 분석 + 단계 점수 로직 조합. Whisper는 "제3자 클라우드 서비스로 개발·구현된 경우"(§III.1.다, p.17)에 해당. AI가 "질병 진단·예측"을 직접 수행하는지 여부가 적용 범위 결정.
  - AQ 자동 계산·단계 자동 판정이 "진단 보조(Impression)"에 해당하면 **MLMD 적용**.
  - 단순 "점수 기록·치료사 참고"이면 **MLMD 회피 가능** (§III.1 "정보제공·관리·모니터링" 품목).

**핵심 요구사항** (적용 시)
- §III.1 성능 기재: 민감도/특이도, 작용원리 주요기능, 훈련데이터 정보(수집시기·기관·규모), 업데이트 예상주기, 클라우드 서비스 종류.
- §III.2 성능·임상적 유효성: 민감도·특이도·PPV·NPV·AUC (단 치료 목적 SaMD는 이 지표가 그대로 안 맞음 — PDF #7 DTx 기준 병행).
- §III.3 임상적 유효성: 전향적·후향적 연구 설계. 시험 데이터셋은 훈련 데이터셋과 독립적이어야 함.
- §III.5 변경허가 대상: "규칙 기반 → 딥러닝 알고리즘 변경", "분석대상·기법 변경" 등.
- §III.6 훈련데이터셋 관리: 관리원칙·조직·QMS 프로세스 수립 필수.

**현재 상태 vs 요구**
| 항목 | 상태 |
|---|---|
| AI 모델 성능평가 인프라 | △ 부분 (evaluationSamplesDb.ts·measured-only 분리 구현, 실측치 없음) |
| 훈련데이터셋 관리정책 | ✗ 미충족 (Whisper는 사외, LLM 프롬프트 로직 관리만) |
| 편향(Bias) 분석 | ✗ 미충족 (60~80대 노인 대상 Whisper 정확도 인종·연령 편향 분석 없음) |
| 전향적/후향적 임상시험 | ✗ 미충족 |
| 클라우드 EaaS 서비스 기재 | △ 부분 (OpenAI API 사용 사실 존재, 공식 기재 없음) |

**필요 조치**
- **T-AI-001** Whisper STT의 역할 공식 정의: "진단 보조"인가 "음성 전사 도구"인가. 전자면 MLMD, 후자면 회피. (담당: E 규제·특허팀 + B 음성·신호팀)
- **T-AI-002** AI 성능 실측 수행: 현 `evaluationSamplesDb.ts` 인프라에 60~80대 대상 최소 30건 세션 데이터 입력. 측정 항목 = 단어·음절 수준 WER, AQ 계산 재현성. (담당: B 음성·신호팀)
- **T-AI-003** 편향 분석 보고서: 연령대(60~80대)·성별·실어증 하위유형 간 성능 편차 분석 (ISO/IEC TR 24027 참조). (담당: B 음성·신호팀)
- **T-AI-004** 훈련데이터셋 관리 정책 문서: Whisper는 외부모델이므로 "재학습 없음·버전 고정" 정책 명시. 내부 파인튜닝 가능성 차단. (담당: A 코드팀 + E 규제·특허팀)

- **T-REG-002** 식약처 사전 분류 문의 (Pre-Submission Meeting): "독립형 디지털의료기기소프트웨어 — 정보제공·관리 유형" vs "디지털치료기기" 분기에 대한 공식 답변 요청. KTL 공인성적서 사전 문의(2026-04-24 발송)와 병행. (담당: E 규제·특허팀)

## 1-4. 가상융합기술이 적용된 디지털의료기기 허가·심사 가이드라인 (PDF #4, 안내서-0872-03, 2025.05)

## 1-5. 디지털의료기기 GMP 가이드라인 (PDF #5, 안내서-1457-01, 2025.11)

**범위 요약**: 「디지털의료제품법」 제24조 "디지털의료기기소프트웨어 품질관리기준 적합판정"의 상세 해석. **허가 전 GMP 심사 필수**. 적합판정 유효기간 3년, 정기심사(§시행규칙 제25조·제35조). 구성: [별표1] 용어정의 / [별표2] 적합판정 일반심사기준 / **[별표3] 디지털의료기기소프트웨어 심사기준** / **[별표4] AI 제어조치 심사기준** / [별첨1~5] SOP 샘플.

# 6. 대표님께 우선 확인받아야 할 의사결정 3가지

1. **DTx 트랙을 갈 것인가, SaMD "정보제공·관리" 트랙으로 갈 것인가?**
   - DTx 트랙: 임상시험 확증 필수 → 6~12개월 추가 + 수천만~억 단위 예산. 대신 "치료제" 포지셔닝·보험수가 가능성.
   - SaMD 정보제공 트랙: 임상시험 면제 가능 → 3~6개월 단축. 대신 "측정·보조 도구" 포지셔닝. 마케팅 메시지 제약.
   - **결정 기준**: 사업모델 — 병원 판매용인가 vs 처방형 DTx인가.

2. **Whisper STT를 "AI 구성요소"로 공식 기재할 것인가, "외부 음성 전사 API"로 축소할 것인가?**
   - 전자 선택 시 → PDF #2 (AI 가이드라인) + GMP [별표4] AI 제어조치까지 full scope. 편향·훈련데이터·AI 방침 전체 문서화.
   - 후자 선택 시 → SOUP(상용 외부 소프트웨어)로 처리, [별표4] 면제 가능. 단 "AI 기반" 마케팅 메시지·특허 출원 시 모순 위험.
   - **결정 기준**: 제품 차별화 포인트 — AI 맞춤형 재활이 가치제안이면 전자 불가피.

3. **GMP 적합판정을 자체 구축할지, 품질관리 위탁(CMO·외주 QMS)을 병행할지?**
   - 자체 구축: 박지수 혼자 품질매뉴얼+5개 SOP+품질조직 모두 작성·운영. 6~12개월.
   - 외주 QMS 컨설팅 병행: 예산 추가(수천만원), 기간 3~4개월 단축. 식약처 심사 경험자 위촉.
   - **결정 기준**: 예산 + 박지수 인력 부담 수용 가능성. KTL 사전문의 회신 결과에 따라 규제 복잡도 확인 후 결정.
