## 1. 적용 방식

이 문서는 NIDS 상담노트를 제품 판정서처럼 쓰지 않는다. NIDS 상담노트는 어떤 가이드라인을 봐야 하는지 알려준 참고자료로 사용한다. 실제 판단은 아래 순서로 한다.

1. 제품제안서와 현재 코드로 브레인프렌즈의 실제 기능과 사용목적을 확인한다.
2. 분류·등급 가이드라인 기준으로 디지털의료기기 해당 여부를 판단한다.
3. SaMD/SiMD 여부와 제품 코드 생성 기준을 확인한다.
4. 등급 결정 기준에 따라 자체 등급 선언 초안을 작성한다.
5. 최종 등급은 식약처/심사기관 검토에서 확정된다는 전제를 유지한다.

## 6. 남은 확인

- 개정안(최종)과 현행 가이드라인 사이의 차이 확인
- 제품 코드 후보를 실제 코드 체계에 맞춰 정리
- `치료·재활` vs `정보제공/관리` 중 어느 축으로 신청할지 RA/식약처 질의로 확정
- 환자 상태를 `심각하지 않음`으로 볼 수 있는지 근거 보강
- IEC 62304 SW 안전성 등급과 오작동 피해 수준을 risk-management-file과 연결

## 7. 분류·등급 → GMP → 개발 증빙 연결

분류·등급 판단은 문서상 선언으로 끝나지 않는다. 브레인프렌즈가 `독립형 디지털의료기기소프트웨어(SaMD)`, `2등급 가능성`, `치료·재활 보조` 축으로 준비된다면 GMP 준비범위와 개발 증빙은 아래처럼 따라온다.

| 분류·등급 판단 | GMP 준비범위 | 개발 증빙 | 연결 요구사항 |
| --- | --- | --- | --- |
| 독립형 SaMD 후보 | 디지털의료기기 SW 심사기준 적용 | SRS, SDS, traceability, V&V export, release manifest | CLASS-005, CLASS-008, GMP-004~006, GMP-009 |
| AI/STT/영상분석 보조기술 포함 | AI/ML 기능을 유지하는 경우 별도 모델/알고리즘 사양과 성능평가 필요 | STT 엔진/모델 버전, 안면/시선 알고리즘 버전, AAC 규칙엔진 버전, WER/CER/RTF, 알고리즘 설명서 | CLASS-003, GMP-027~029 |
| 치료·재활 축으로 보수 적용 | 품질시스템, 위험관리, 검증완료, 잔여위험, 릴리스 승인 기록 필요 | release readiness, residual risk approval, change request, CAPA, PMS | CLASS-010~012, GMP-015~025 |
| 오작동 시 간접 피해 가능 | 위험관리와 치료사 검토 통제가 필수 | target/transcript/audio 보존, reviewRequired, 치료사 검토 로그, 보호자 제한문구 | CLASS-013~015, GMP-015, DYS-023~025 |
| 2등급 가능성 기준 준비 | 제조업 허가 → GMP 적합인정 → 품목 인허가 순서로 실행 | GMP/QMS 결정표, 품질책임자, 외주/하이브리드 결정, 심사 대응 패키지 | CLASS-016, GMP-002~003, NIDS-001~006 |

### 7.1 PM 판단

현재 개발팀이 해야 할 일은 “등급을 확정했다”고 주장하는 것이 아니라, 2등급 SaMD로 보아도 방어 가능한 증빙 구조를 만드는 것이다. 따라서 다음 개발은 아래 순서로 고정한다.

1. 결과 저장/ZIP export에 원자료, 점수근거, 앱/모델/문서 버전을 포함한다.
2. release manifest를 GMP 출시기록으로 확장한다.
3. 문제보고/CAPA export를 만들어 STT 실패, 저장 실패, 채점 이상을 추적한다.
4. 치료사 검토 화면에서 원음, target, transcript, 정오답, reviewRequired를 확인한다.
5. 마비말장애 DTx 트랙용 30개 목표단어/PCC, MPT, DDK, 말 명료도 수동평가를 준비한다.
