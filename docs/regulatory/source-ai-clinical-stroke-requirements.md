# Source AI Clinical Trial Plan Requirements Matrix

작성일: 2026-05-08  
원문: `인공지능기술이+적용된+디지털의료기기의+임상시험계획서+작성+가이드라인(민원인+안내서)+허혈성+뇌졸중.pdf`  
목적: 인공지능기술이 적용된 디지털의료기기의 후향적 임상시험계획서 작성 원칙을 브레인프렌즈 AI/STT/채점/안면·시선 평가 준비사항으로 분해한다.

## 1. 해석 원칙

- 원문 예시는 `허혈성 뇌졸중 보조진단 AI`와 `후향적 임상시험` 중심이다.
- 브레인프렌즈는 현재 진단보조 AI가 아니라 언어재활 보조 SaMD이므로, 원문을 그대로 적응증에 적용하지 않는다.
- 다만 AI 기능의 임상적 성능 검증, 데이터셋 독립성, 참조표준, 통계분석, 데이터 보안, 임상시험용 SW 관리 원칙은 브레인프렌즈 AI/STT 평가에 직접 참고한다.
- 자동 진단/치료 결정이 아니라 치료사 검토용 보조 지표라는 경계는 유지한다.

## 2. 브레인프렌즈 적용 결론

브레인프렌즈에서 이 문서가 요구하는 핵심은 “AI가 맞다/틀리다”를 주장하려면 아래가 분리되어야 한다는 점이다.

1. 학습/개발 데이터와 시험 데이터셋의 독립성
2. 참조표준 또는 gold label 작성자와 평가자 역할 분리
3. 양성/음성 또는 성공/실패 분포와 수집기간 명시
4. 민감도, 특이도, AUC, 일치율, WER/CER 등 지표와 성공기준 사전 정의
5. 데이터셋 손실, 개인정보 노출, 모델 오류로 인한 안전성/위험관리

## 3. 요구사항 매트릭스

| ID | 원문 요구사항 요지 | 브레인프렌즈 현재 상태 | 해야 할 일 |
| --- | --- | --- | --- |
| AICLIN-001 | 후향적 임상시험은 기존 의료용 데이터를 이용하므로 시험 데이터셋 수집방법, 유효성 평가변수 등을 고려해 설계해야 한다. | AI 평가셋 계획 수준 | STT/채점/안면·시선 평가셋의 수집기간, 출처, 포함/제외 기준 정의 |
| AICLIN-002 | 임상시험계획서에는 제목, 기관, 책임자, 관리자, 의뢰자, 목적, 기기사항, 대상자/데이터셋, 방법, 관찰항목, 유효성/안전성, 동의/보상/안전보호 등이 포함된다. | 임상계획서 없음 | `clinical-protocol-outline.md`에 AI 평가 섹션 추가 |
| AICLIN-003 | 임상시험 제목은 목적, 방법, 디자인을 구체적으로 드러내야 한다. | 없음 | “한국어 언어재활 과제 STT/채점 보조지표 후향적 성능평가” 식의 내부 연구 제목 정의 |
| AICLIN-004 | 임상시험기관은 지정 기관 또는 관련 법상 허용 기관이어야 한다. | 임상 협력기관 미확정 | 부산대병원 등 협력기관 역할과 데이터 접근 권한 정리 |
| AICLIN-005 | 시험책임자, 담당자, 판독자/참조표준 작성자, 통계전문가, 의료기기 전문가를 포함해야 한다. | 역할표 일부 | 치료사/SLP gold label 작성자, AI 평가 담당자, 통계 자문 역할표 필요 |
| AICLIN-006 | 임상시험용 디지털의료기기 관리자를 지정해야 한다. | 웹앱 배포 중심 | 평가용 앱 버전/모델 버전 관리자와 접근권한 관리 로그 지정 |
| AICLIN-007 | 의뢰자는 임상시험 계획·관리·재정 책임자이며 모니터요원/CRO 정보를 기재한다. | 미정 | 외부 시험기관 또는 CRO를 쓸 경우 공급자/위탁 관리표 추가 |
| AICLIN-008 | 목적/배경에는 안전성·유효성, 적응증 현황, 표준진단법 성능, 개발경위, 작동원리, 분석적 성능을 포함한다. | AI 허가 대응표 있음 | 브레인프렌즈는 표준치료/표준평가 대비 보조지표 목적과 선행연구를 분리 기술 |
| AICLIN-009 | 분석적 성능은 데이터 출처, 규모, 양음성 비율, 시험결과(AUC, 민감도, 특이도 등)를 구체적으로 기재한다. | 실측 부족 | WER/CER/RTF, target phrase 일치율, 치료사 판정 일치율, 양/음성 분포 저장 |
| AICLIN-010 | 사용목적은 사용자, 대상질환/적응증, 입력정보, 분석기술, 출력정보를 구체적으로 적는다. | claim-lock 있음 | STT/안면/시선/AAC/채점별 입력·처리·출력표 작성 |
| AICLIN-011 | 작용원리는 AI 모델 구조, 학습 데이터, 출력값 산출 방식을 설명한다. | 모델 사양표 없음 | OpenAI STT, 규칙기반 AAC, 안면/시선 랜드마크, 채점 알고리즘 사양표 작성 |
| AICLIN-012 | 후향적 임상시험에서는 대상자를 시험 데이터셋으로 대체하며 엄격한 선정/제외 기준을 제시해야 한다. | 평가셋 기준 없음 | 녹음 품질, 언어과제, 연령, 중증도, 소음, 기기 조건의 포함/제외 기준 정의 |
| AICLIN-013 | 데이터셋 수는 가설, 유의수준, 검정력, 통계방법, 예상 효과 차이로 산정해야 한다. | 10~30개 내부 예비시험 수준 | 예비시험은 탐색용으로 표시하고, 확증 성능평가 전 샘플 수 산정 필요 |
| AICLIN-014 | 정상/비정상군 또는 양성/음성군의 구분과 비율을 고려해야 한다. | 양/음성 분포 없음 | 발화 성공/실패, 정확/오류, clear/noisy, aphasia/dysarthria 등 stratification |
| AICLIN-015 | 참조표준 구성을 위한 정보가 있어야 한다. | 치료사 검토 미흡 | gold transcript, target, SLP 정오답, 말 명료도 수동평가를 참조표준으로 설계 |
| AICLIN-016 | 입력 정보의 임상적·기술적 표준을 고려해야 한다. | 녹음 포맷 저장 문제 해결 중 | sample rate, codec, mic type, browser, noise level, task ID를 metadata로 저장 |
| AICLIN-017 | 시험 데이터셋은 개발·학습 데이터와 독립성을 유지해야 한다. | 미분리 | 개발용 샘플, 튜닝용 샘플, 시험용 locked set을 물리적으로 분리 |
| AICLIN-018 | 무작위배정과 눈가림 전략을 통계 전문가와 상의해 포함해야 한다. | 없음 | 평가자에게 AI 결과를 숨긴 상태에서 SLP gold label 작성 프로세스 정의 |
| AICLIN-019 | 참조표준 작성자와 판독자/평가자는 독립적으로 구성해야 한다. | 치료사 화면 미완 | gold label 작성자와 앱 결과 검토자를 구분하는 운영 규칙 작성 |
| AICLIN-020 | 시험 데이터셋 배정과 강화배정 등은 목적에 맞춰 기술해야 한다. | 없음 | 발화 오류/정상 발화 비율을 의도적으로 맞춘 평가셋 구성 계획 |
| AICLIN-021 | 임상시험용 디지털의료기기의 표준적인 사용방법을 자세히 기술해야 한다. | 사용 흐름 있음 | step-2/4/5/sing-training별 평가 절차와 입력 순서 문서화 |
| AICLIN-022 | 관찰항목에는 인구학적 정보, 선정/제외 판정, 참조표준, 기기 적용 전후, 유효성 평가가 포함된다. | metadata 일부 | age group, severity, diagnosis, device, task, gold label, model output를 export |
| AICLIN-023 | 후향적 시험은 환자 직접 부작용은 낮지만 의료기기 오류로 인한 오진/오판 위험이 있을 수 있다. | 위험관리파일 일부 | “AI/STT 오류로 치료사 판단 보조자료가 왜곡” 위험항목 추가/보강 |
| AICLIN-024 | 입력 데이터는 최소사양 이상 품질이어야 하고 개인정보 노출을 방지해야 한다. | 품질/비식별화 일부 | 녹음/영상 품질 기준, 식별자 제거, 파일명 가명화, 업로드 전 검증 |
| AICLIN-025 | 시험 데이터셋 손실·파괴 또는 소프트웨어 결함으로 임상시험 중지가 가능하다. | 백업/복구 미흡 | 평가셋 백업, hash 검증, 손상 감지, 재실행 불가 기준 정의 |
| AICLIN-026 | 유효성 평가는 1차/2차 평가변수, 통계방법, 성공기준, 대조군 유형과 근거를 제시해야 한다. | 기준 미정 | STT는 WER/CER, 채점은 SLP 일치율, 안면/시선은 결정성/품질지표로 성공기준 정의 |
| AICLIN-027 | 성공기준은 업체가 설정할 수 있으나 설정 사유와 근거를 제시해야 한다. | 목표치만 있음 | WER 15% 등 정량 목표는 실측 전 “목표”로만 두고 근거 문서화 |
| AICLIN-028 | 안전성 평가는 소프트웨어 오류로 인한 오진/오판과 관련된 평가기준·방법·보고방법을 제시해야 한다. | 문제보고/CAPA 필요 | AI 오류/오판/저장실패를 AE/ADE와 문제보고로 분리 기록 |
| AICLIN-029 | 이상사례 인과관계는 Definitely/Probably/Possibly/Probably not/Definitely not/Unknown 등으로 평가한다. | adverse event 기능 일부 | AE 관련성 등급 필드 추가 |
| AICLIN-030 | 중대한 이상반응은 7일/8일 추가/15일 등 정해진 기한 내 보고한다. | 보고기한 필드 없음 | SAE/ADE 보고기한 추적 필드와 알림 필요 |
| AICLIN-031 | 후향적 시험에서 동의 면제는 IRB 승인에 따라 고려할 수 있다. | IRB 문서 일부 | 평가셋 사용 시 동의/면제 근거와 IRB 승인 상태를 기록 |
| AICLIN-032 | 데이터셋 활용 시험은 환자정보 보호 의무가 중요하다. | 보안 문서 있음 | subject code, 재식별 분리, 접근권한, 열람 로그 강화 |
| AICLIN-033 | 임상시험계획서 변경은 IRB/MFDS 변경승인이 필요할 수 있고 개정일자·이유·내용을 기록해야 한다. | 변경관리 SOP 있음 | 임상/AI 평가 프로토콜 변경을 change request와 연결 |
| AICLIN-034 | 임상 관련 기록은 보안 유지하며 시험 완료 후 3년 보존한다. | 보존기간 미확정 | AI 평가셋/결과/CRF/모델출력 보존기간 정책 추가 |
| AICLIN-035 | 임상시험용 디지털의료기기는 승인 전 공급 금지, 취급·보관·반납·폐기·회수 기록이 필요하다. | 웹 배포 중심 | 평가용 앱 버전 배포/권한 회수/모델 폐기 로그 설계 |

## 4. 브레인프렌즈에 직접 반영할 개발 항목

| P0 | gold label / SLP 판정 / 앱 출력 분리 저장 | 참조표준과 AI 결과 독립성 확보 |
| P0 | 학습/튜닝/시험 데이터셋 분리 | 개발 데이터와 시험 데이터 독립성 요구 |

| P0 | AI 오류/오판 문제보고 | 소프트웨어 오류로 인한 안전성 평가 대응 |
| P1 | 통계분석 계획 | WER/CER, 일치율, 민감도/특이도 등 성공기준 사전 정의 |
