# KTL 방문 미팅 준비 — 공인성적서 확보 전략

작성일: 2026-05-11

대상기관: 한국산업기술시험원(KTL)

## 1. 한 줄 결론

브레인프렌즈는 `2등급 가능성 기준 독립형 SaMD`로 준비 중이며, KTL 방문에서는 **(1) 어떤 시험 항목으로 공인성적서를 받을 수 있는지 범위 확정, (2) KTL이 디지털의료제품법 제2026-5호상 수탁(시험·성능검사) 기관으로 지정되어 있는지 확인, (3) 시험 진행을 위한 사전 제출자료·일정·비용을 확정**하는 것이 목표다.

## 2. KTL에서 받을 수 있는 공인성적서 후보 (제24조 첨부서류와 매핑)

| 시험 트랙 | 근거 | 제24조 대응 자료 | 우리 준비 상태 |
| --- | --- | --- | --- |
| ① SW 검증 및 유효성(V&V) | IEC 62304 / 고시 제2026-6호 제24조 3호, 제26조 | SRS, SDS, traceability, 단위/통합/시스템 시험, anomaly/retest/impact | 일부 완료 (SRS/SDS/traceability 있음, anomaly/retest/impact 부족) |
| ② 사이버보안 | 식약처 「의료기기 사이버보안 허가·심사 가이드라인」 35개 요구사항 / 제24조 5호 | IA/UC/SI/DC/TRE/RA 35개 항목 대응표 + 시험 결과 | 약 77% 일치(내부 준비율). 백업/복구·권한표·외부 침투시험 부족 |
| ③ 사용적합성 | IEC 62366-1:2015 + AMD1:2020 / 제24조 6호 | POF·critical task·use scenario·formative/summative 결과 | 프로토콜 완성, 실제 수행 결과 부족 |
| ④ 성능(분석성능) 시험 | 제2026-6호 제25조 갈음 전략(임상시험 자료 일부 갈음) | locked test set, SLP gold label, WER/CER/RTF/PCC | 부족 — locked set·gold label 구조 잠금 필요 |
| ⑤ 위험관리 검토 | ISO 14971 / 제24조 1호·8호 | Risk management file, 잔여위험 승인 | 일부 완료 (RM-* 매핑 있음, 신규 위험 갱신 필요) |

> 우선순위 권고: **① SW V&V → ② 사이버보안 → ③ 사용적합성 → ④ 분석성능** 순으로 시험 트랙을 묶고, ⑤ 위험관리는 ①~④ 결과를 흡수하는 흐름으로 가는 게 안전함.

## 3. KTL 측에 반드시 확인해야 할 7가지

| Q# | 질문 | 답을 받아야 하는 이유 |
| --- | --- | --- |
| Q1 | 디지털의료제품법 시행 후 고시 제2026-5호 기준으로 KTL이 **성능검사·시험·인증업무 수탁기관으로 지정**되어 있는지(품목/시험 종류별 범위 포함) | 비지정 기관 시험성적서는 제24조 첨부서류로 인정되지 않을 수 있음 |
| Q2 | 독립형 SaMD에 대해 IEC 62304 SW V&V 시험성적서 발급이 가능한지, 발급 범위(요구사항/설계/단위/통합/시스템) | 우리는 제24조 3호 대응으로 공인성적서가 필요 |
| Q3 | 식약처 「의료기기 사이버보안 가이드라인」 35개 요구사항 기준 시험 + 침투시험 발급 가능 여부 | 제24조 5호 대응. 외부 침투시험은 우리 내부 부족 항목 1순위 |
| Q4 | IEC 62366-1 사용적합성 평가의 **위탁 수행 + 공인성적서 발급** 가능 여부, formative/summative 분리 가능 여부, 5/5/5 참가자 모집 지원 가능 여부 | 사용적합성은 자체 수행 가능한지 위탁이 권고되는지 명확하지 않음 |
| Q5 | 제25조 분석성능 검증으로 임상자료 일부를 갈음하려는 전략에 대해, KTL이 **분석성능 시험성적서**(WER/CER/RTF/PCC 등)를 발급할 수 있는지 | 우리 1차 인허가 전략 핵심 |
| Q6 | 시험 신청 전 **사전검토(pre-test) 또는 적합성 회의** 절차가 있는지, 있다면 비용/일정/필요자료 | 기준 미충족 상태로 본시험 들어가면 비용·기간 모두 손해 |
| Q7 | 시험 트랙별 **표준 일정·비용·필요 산출물 체크리스트**, 그리고 GMP 적합인정 전에도 시험 접수가 가능한지 | NIDS 상담노트 기준 우리 순서는 제조업 허가 → GMP → 품목 인허가. 시험 접수 시점 의사결정 필요 |

## 4. KTL 방문 전 우리가 가져갈 자료

### 4.1 인쇄/USB로 지참 (데이터팩)

1. **제품 개요 1장** — 사용목적, 대상환자, 사용자군(환자/치료사/보호자), 제품 유형(독립형 SaMD), 준비 등급(2등급 가능성)
2. **`intended-use-and-contraindications.md`** — 사용목적·대상환자·금기
3. **`ai-role-boundary.md`** — AI는 치료사 검토용 보조지표라는 경계
4. **`claim-lock.md`** — 허용/금지 문구 (KTL 측 오해 방지)
5. **`srs.md`, `sds.md`, `traceability-matrix.md`** — IEC 62304 산출물
6. **`risk-management-file.md`** — ISO 14971 RM-* 항목
7. **`source-cybersecurity-requirements.md`** — 35개 사이버보안 요구사항 대응표
8. **`usability-evaluation-protocol.md`** — IEC 62366 프로토콜 + POF/critical task
9. **`article-24-dossier-index.md`** — 제24조 첨부서류 8종 현재 상태표
10. **`cloud-and-data-transfer.md`** — STT 서버 보안 프록시, 데이터 전송 정책

### 4.2 시연 가능한 것 (요청 시)

- BrainFriends 웹앱 데모 (환자/치료사 화면)
- 결과 ZIP export 샘플 (audio, target, transcript, scoreReason 포함된 구조)
- release manifest 샘플 (SBOM, SOUP)
- 보안 감사로그 샘플

## 5. 미팅 자리에서 우리가 먼저 정리해서 말할 포인트

1. "**독립형 SaMD, 2등급 가능성 기준**으로 준비 중. 최종 등급은 식약처 사전검토 결과를 따른다."
2. "AI/STT/안면/시선/AAC는 **자동 진단·처방이 아니라 치료사 검토용 보조지표**다. UI·로그·문서 모두 동일한 경계로 잠가 두었다."
3. "STT는 **완전 온디바이스 클레임을 폐기**하고 **서버 보안 프록시 기반**으로 정리했다. 데이터 전송·저장·보존 정책은 별도 문서로 잡혀 있다."
4. "1차 인허가 전략은 **제25조의 분석성능 검증으로 임상자료 일부 갈음**을 노린다. 따라서 KTL의 분석성능 시험 가능 여부가 핵심이다."
5. "GMP/QMS는 **하이브리드(코어 자체 + 심사 대응 외주)** 방향이며, 제조업 허가 → GMP → 품목 인허가 순서로 간다. 시험 접수 시점은 KTL 권고를 듣고 확정하고 싶다."
6. "치료효과·DTx·온디바이스·자동 진단/처방 같은 **확정 표현은 식약처 인정 전까지 사용하지 않는다.** 계약·견적·성적서 표현도 동일 기준 부탁한다."

## 6. 절대 사용 금지 표현 (`claim-lock.md` 기준)

- "2등급 확정"
- "허가 완료 / 인증 완료 / GMP 적합인정 완료"
- "DTx 허가 제품"
- "치료효과 입증 / WER 15% 이하 달성"
- "완전 온디바이스 STT"
- "AI 자동 진단 / 처방 / 치료 결정"
- "임상시험 면제 확정"

이 문구들은 시험 신청서, RFP, 견적서, 성적서 어디에도 들어가면 안 된다.

## 8. 한 문장 메모

KTL 미팅의 핵심은 "이미 준비된 문서로 어디까지 시험 접수가 가능한지"를 듣는 자리이지, "성적서를 미리 받겠다"는 자리가 아니다. **사전검토(Q6)와 수탁기관 지정 범위(Q1)에서 답을 못 받으면, 그날의 다른 모든 답은 가치가 떨어진다는 점**을 잊지 않는다.
