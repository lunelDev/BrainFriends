# Test And Certification Development Guide

작성일: 2026-05-07  
기준: `mfds-guideline-basis.md`  
목적: 브레인프렌즈를 기능 위주가 아니라 식약처 시험·인증·심사 기준에 맞춰 개발하기 위한 실무 기준서

## 1. 결론

브레인프렌즈는 이제 "새 기능을 많이 넣는 제품"이 아니라 "시험과 심사에서 설명 가능한 제품"으로 개발해야 한다.

개발 우선순위는 아래 순서로 고정한다.

1. 저장/추적성
2. 채점/판정 신뢰성
3. 치료사 검토 가능성
4. 보안/권한/로그
5. 임상시험과 성능평가에 바로 쓸 수 있는 데이터 구조

## 2. 가이드라인별 개발 요구사항

| 가이드라인 | 개발에서 맞춰야 하는 것 | 브레인프렌즈 적용 |
| --- | --- | --- |
| 디지털의료기기 분류·등급 지정 개정안 | 제품 사용목적, 위해 수준, 오작동 영향이 코드/문서/UI에 일관돼야 함 | 자동 진단/치료 결정 제거, 재활 보조 SaMD로 고정 |
| 디지털의료기기 GMP 가이드라인 | 설계문서, 시험기록, 변경관리, 완제품 시험, 공급업체 관리가 남아야 함 | 결과 저장, release manifest, 테스트 로그, SOP 연결 |
| AI 허가·심사 가이드라인 | 시험 데이터셋, 참조표준, 정확도 기준, 업데이트 주기, 데이터 출처를 설명 가능해야 함 | transcript, gold label, 모델 버전, 평가셋 버전 보존 |
| 사이버보안 허가·심사 가이드라인 | IA~RA 35개 요구사항에 대응하는 입력검증, 권한통제, 감사로그, 세션통제가 필요 | 현재 보안 코드 + 치료사/환자/보호자 권한 검증 강화 |
| 마비말장애 DTx 안전성·성능/임상시험계획서 가이드라인 | 말 명료도, MPT, DDK, PCC, PHQ-9, QoL-Dys, 안전성 변수, 사용환경, 치료 순응도 자료가 필요 | 원음/target/transcript/정오답, 30개 목표단어 PCC, MPT/DDK 과제, 완료율, 중단 사유, 이상반응 수집 |
| AI 임상시험계획서 가이드라인 | 시험 목적, 대상자 기준, 데이터 규모, 양/음성 분포, 참조표준, 독립 시험셋, 분석 방법을 재현 가능하게 남겨야 함 | 환자별 세션 결과, 녹음, transcript, SLP gold label, metadata, locked test set, export 구조 정리 |

## 3. 제품에서 반드시 남겨야 하는 증적

### 3.1 결과 저장

- 원문 transcript
- target phrase
- 채점 결과와 세부 점수
- 녹음 파일 경로 또는 첨부 파일
- 안면/시선/AAC 사용 여부
- 측정 품질 상태
- reviewRequired 여부
- STT 엔진/모델 버전

### 3.2 사용 로그

- 훈련 시작
- 훈련 중단
- 녹음 실패
- 마이크/카메라 권한 거부
- 저장 실패와 재시도
- 치료사 검토 완료
- 보호자 링크 조회

### 3.3 시험용 메타데이터

- 세션 ID
- 환자 가명 ID
- 자극어/문항 ID
- step 종류
- 디바이스 종류
- 브라우저/앱 버전
- 모델 버전
- 결과 생성 시각

## 5. 개발 금지 방향

- 온디바이스 STT를 기본 기능처럼 계속 확장
- 임상 전 치료효과를 전제로 한 UI/문구 추가
- 자동 판정만 보여주고 근거 데이터를 저장하지 않는 기능
- 로그 없이 결과만 남기는 기능
- 제품 범위를 넓히는 새로운 적응증 추가

## 6. 바로 실행할 개발 체크리스트

1. step-2, step-4, step-5 결과에 `target`, `transcript`, `scoreReason`, `reviewRequired`를 공통 저장
2. 결과 ZIP에 `audio`, `transcript`, `result.json`, `history.json`이 빠지지 않게 통일
3. 치료사 화면에서 문제별 원문/정오답/녹음 재생 가능하게 표시
4. `training-events`에 권한거부, 저장실패, 중단 이벤트 추가
5. AI 평가용 CSV export 스키마를 고정

## 7. 한 줄 PM 기준

브레인프렌즈의 다음 개발은 "얼마나 똑똑해 보이느냐"가 아니라 "시험자료와 심사자료로 바로 전환 가능한가"를 기준으로 결정한다.
