# AI 허가·심사 방안 대응표

작성일: 2026-05-07  
기준: `인공지능기술이 적용된 디지털의료기기의 허가·심사 가이드라인(민원인 안내서)` III. 허가·심사 방안  
목적: 브레인프렌즈의 AI/디지털 기능이 허가·심사 문서에서 어떤 항목으로 설명되어야 하는지 고정한다.

## 1. 적용 범위

브레인프렌즈는 독립형 디지털의료기기소프트웨어(SaMD)로 준비한다. AI 또는 디지털 기술이 포함된 기능은 아래 항목별로 설명한다.

- 서버 보안 프록시 STT
- 안면 랜드마크 기반 발화/구강 움직임 보조지표
- 시선 보조지표
- AAC 입력
- 자가진단/언어재활/노래훈련 결과 저장
- 치료사 검토용 결과 표시

AI 결과는 진단, 처방, 치료 결정을 자동 수행하지 않는다. 모든 결과는 치료사 검토용 보조 지표로 제한한다.

## 2. 성능(기능) 또는 특성 기재 6개 항목

| 가이드라인 항목 | 브레인프렌즈 기재 내용 | 현재 상태 | 개발/문서 보완 필요 |
| --- | --- | --- | --- |
| 1. 임상적 관점에서의 성능 | 언어재활 과제 수행 결과, STT 전사 정확도(WER/CER), 반응시간, 과제별 정오답, 치료사 검토 일치도, 오류/재시도율로 기재한다. | 일부 구현 | 과제별 점수 산출 근거, transcript, 녹음 파일, reviewRequired 저장을 고정해야 함 |
| 2. 작용원리에 기반한 세부 프로그램의 주요 기능 | 음성 입력을 STT로 전사하고, 안면/시선/AAC/과제 수행 정보를 함께 저장하여 치료사 검토 화면과 결과 리포트에 제공한다. | 일부 구현 | step-2/4/5, sing-training, self-assessment 저장 구조를 통일해야 함 |
| 3. 적용되는 디지털 기술의 세부적 특징 | STT, MediaPipe 안면 랜드마크, 시선 보조지표, AAC 규칙 기반 입력, 결과 저장/export, 치료사 검토 UI를 모듈별로 기재한다. | 일부 문서화 | 모델명, 버전, 실행 위치, 입력/출력 형식을 release manifest와 연결해야 함 |
| 4. 학습데이터의 정보 | 수집 시기, 수집 기관, 대상자 특성, 샘플 수, 음성/영상/텍스트 구분, 라벨링 기준, 학습/검증/시험 분리 원칙을 기재한다. | 부족 | 내부 예비시험 데이터셋 표와 향후 임상 수집 데이터셋 표가 필요함 |
| 5. 학습데이터의 업데이트 예상 주기 | 제조업체 임의 상시 업데이트가 아니라 변경관리 SOP에 따른 반기 또는 연간 재평가 후 반영으로 기재한다. | 부족 | 변경관리 SOP에 AI/STT/평가셋 업데이트 주기를 명시해야 함 |
| 6. 클라우드 서비스 종류 | 브라우저, 자사 서버, 외부 STT 서비스, DB/스토리지의 역할과 전송/저장/보존 범위를 기재한다. | 일부 문서화 | 클라우드 사업자, 저장 위치, 보존 기간, 제3자 처리 여부 표가 필요함 |

## 3. 허가 문서에 넣을 초안 문구

### 3.1 임상적 관점에서의 성능

본 제품은 실어증 또는 마비말장애 환자의 언어재활 훈련 수행과 경과 모니터링을 보조하기 위해 과제별 수행 결과, 음성 전사 결과, 반응시간, 안면/시선 보조지표, AAC 사용 이력을 수집한다. 성능은 자동 진단 정확도가 아니라 치료사 검토를 보조하기 위한 전사 정확도, 과제 수행 일치도, 결과 저장 완전성, 오류/재시도율, 치료사 검토 일치도 기준으로 평가한다.

### 3.2 작용원리 기반 주요 기능

사용자는 자가진단, 언어재활, 노래훈련 과제를 수행한다. 제품은 음성, 안면 움직임, 시선 상태, AAC 입력, 과제 응답을 세션 단위로 저장하고, 치료사 화면에서 문제별 결과와 원자료를 검토할 수 있게 제공한다. 최종 평가와 치료 결정은 의료진이 수행한다.

### 3.3 디지털 기술의 세부 특징

| 기술 | 실행 위치 | 입력 | 출력 | 역할 |
| --- | --- | --- | --- | --- |
| STT | 서버 보안 프록시 | 녹음 음성 | transcript, confidence, model metadata | 치료사 검토용 전사 보조 |
| 안면 분석 | 브라우저 | 카메라 영상 | 구강/조음 보조지표 | 발화 수행 참고 지표 |
| 시선 분석 | 브라우저 | 카메라 영상 | attentionRatio, offTaskTimeMs | 측정 품질 참고 지표 |
| AAC | 브라우저/서버 | 심볼 선택 sequence | intent sentence, selectedIds | 발화가 어려운 환자의 대체 응답 |
| 결과 저장/export | 서버/브라우저 백업 | 세션 결과 | JSON/CSV/ZIP | 시험·검토 증적 보존 |

### 3.4 학습데이터 정보

현재 허가 제출용 확정 학습데이터 표는 미완성이다. 제출 전까지 아래 형식으로 확정한다.

| 데이터셋 | 수집 시기 | 수집 기관 | 대상자 | 샘플 수 | 데이터 종류 | 라벨링 기준 | 사용 목적 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 내부 예비 STT 평가셋 | 미정 | 자사/협력기관 | 60~80대 또는 대상군 | 미정 | 음성/전사 | 목표문장, 실제 전사, 치료사 검토 | WER/CER/RTF 산출 |
| 사용성 예비시험 로그 | 미정 | 자사/위탁기관 | 환자/보호자/치료사 | 미정 | 행동 로그/오류 로그 | 과제 성공/실패, 재시도, 중단 사유 | IEC 62366 사용성 평가 |
| 임상 또는 후속 RCT 데이터셋 | 미정 | 임상 협력기관 | 뇌졸중 후 언어장애 대상자 | 미정 | 음성/영상/결과 | 임상 프로토콜 기준 | 성능/안전성 평가 |

## 4. 학습데이터 업데이트 예상 주기

| 변경 유형 | 기본 주기 | 변경관리 기준 |
| --- | --- | --- |
| STT 모델 변경 | 반기 또는 연간 재평가 후 반영 | WER/CER/RTF 기준 재검증, release manifest 갱신 |
| 안면/시선 분석 모델 변경 | 반기 또는 주요 버전 변경 시 | 결정성 V&V, 측정 품질 회귀 테스트 |
| AAC 규칙/어휘 변경 | 분기 또는 콘텐츠 배포 시 | 치료사 검토, 문구/의도 매핑 V&V |
| 평가셋 추가 | 수집 완료 후 잠금 | 데이터셋 버전, 라벨링 기준, 대상자 특성 문서화 |

상시 자동 학습 또는 운영 중 무검증 모델 업데이트는 적용하지 않는다.

## 5. 클라우드 서비스 종류

| 구간 | 서비스 형태 | 처리 데이터 | 저장 여부 | 보완 필요 |
| --- | --- | --- | --- | --- |
| 브라우저 | 환자 앱 | 카메라 영상, 마이크 음성, AAC 선택 | 원칙적으로 로컬 처리 또는 업로드 전 임시 보관 | 브라우저 임시 저장/삭제 정책 명시 |
| 자사 서버 | 보안 프록시/API | 인증 정보, 세션 결과, STT 요청 | DB/스토리지 저장 가능 | 저장 위치, 보존 기간 확정 |
| 외부 STT 서비스 | 제3자 AI API | STT 요청용 음성 또는 파생 오디오 | 원본 보존 금지 또는 최소화 정책 필요 | 위탁/제3자 처리 고지 확정 |
| DB/스토리지 | 결과 보존 | 결과 JSON, transcript, audio file, export | 보존 정책에 따라 저장 | 환자 동의, 삭제 요청, 접근권한 표 필요 |

## 7. 금지 표현

- AI가 진단한다
- 자동 치료한다
- 치료 효과가 입증됐다
- 원본 음성이 전송되지 않는다
- 완전 온디바이스 STT가 기본이다
- 임상시험 없이 치료효과를 보장한다
