# 브레인프렌즈 허가 준비 내부 기준서

작성일: 2026-05-08  
목적: 상담용 제출자료가 아니라, 브레인프렌즈 팀이 개발과 문서 갱신 때 계속 확인할 내부 기준서다. 제품 허가·인증·심사와 직접 관련된 판단, 현재 준비 상태, 부족 항목, 개발 우선순위를 한 문서에서 본다.

| --- | --- |
| 제품 유형 | 독립형 디지털의료기기소프트웨어(SaMD) |
| 준비 등급 | 2등급 가능성 기준 |
| 사용목적 | 실어증·마비말장애 환자의 언어재활 훈련 및 경과 모니터링 보조 |

| 최종 확정 | 식약처 품목분류/사전검토/허가·인증 심사 결과에 따름 |

## 2. 한 줄 결론

브레인프렌즈는 `2등급 가능성 기준 독립형 SaMD`로 준비한다. 다만 이 전략은 기능 수가 아니라 `제24조 첨부서류`, `제25조 분석성능 갈음 가능성`, `제26조 세부 작성요건`, `치료사 검토 증적`, `결과 원자료 저장`, `보안`, `사용적합성`, `GMP/QMS 기록`으로 방어해야 한다.

따라서 당장 개발 우선순위는 새 기능 추가보다 `결과 저장/ZIP`, `release manifest`, `문제보고/CAPA`, `치료사 검토 로그`, `locked test set/gold label` 쪽이다.

## 5. 제25조 면제·갈음 전략

제25조는 자동 면제 조항이 아니다. 식약처가 제품 특성, 사용목적, 임상적 주장, 위험도를 보고 인정해야 한다.

| 전략 | 현재 판단 | 전제 조건 |
| --- | --- | --- |
| 임상시험 자료 전체 제출 | DTx/치료효과 주장 시 필요 가능성 큼 | IRB, 임상기관, 통계계획, 유효성/안전성 endpoint |
| 2등급 SW 분석성능 검증으로 일부 갈음 | 현재 1차 권장 전략 | 원음, target, transcript, scoreReason, locked test set, gold label, WER/CER/RTF/PCC |
| 정보제공·관리 목적 일부 갈음 | 가능하지만 문구 제한 필요 | 치료효과보다 경과 모니터링/보조지표 중심 |
| 우수 관리체계 인증에 따른 자료 면제 | 장기 후보 | 우수 관리체계 인증서 확보 전까지 완료 표시 금지 |

현재는 `치료효과 입증`을 전면 주장하지 않고, `치료사 검토용 보조지표 + 경과 모니터링 보조`로 사용목적을 좁힌 뒤 분석성능 검증 자료를 갖추는 전략이 가장 안전하다.

## 7. GMP/QMS 관점

허가자료는 기능 설명만으로 방어되지 않는다. 디지털의료기기 GMP 관점에서는 요구사항, 설계, 구현, 시험, 변경, 릴리스, 문제보고, CAPA, PMS가 연결되어야 한다.

| GMP/QMS 항목 | 현재 상태 | 필요 증빙 |
| --- | --- | --- |
| SRS/SDS/Traceability | 일부 완료 | 최신 변경분 반영: AAC, 시선, 저장, 노래훈련, STT 정책 |
| SOUP/SBOM | 일부 완료 | 패키지 버전, 취약점, 예외 승인, release manifest 연결 |
| 릴리스 기록 | 부족 | git SHA, package-lock, V&V 결과, 잔여위험 승인 |
| 변경관리 | 초안 | 변경 영향분석, 재시험, 변경허가 필요성 판단 |
| 문제보고/CAPA | 초안 | STT 실패, 저장 실패, 채점 이상, 권한 거부 export |
| PMS | 초안 | 시판 후 이상사례/성능저하/보안이슈 수집 절차 |

## 8. 사이버보안과 데이터 전송

STT는 완전 온디바이스 클레임을 폐기하고 서버 보안 프록시 기반으로 정리했다. 원본 음성, 전사, 결과 저장은 허가와 보안심사에서 설명 대상이다.

| 항목 | 현재 정리 | 부족한 것 |
| --- | --- | --- |
| STT 처리 | 브라우저 → 자사 서버 보안 프록시 → 외부 STT 또는 서버 처리 | 사업자, 저장 위치, 보존기간 확정 |
| 안면/시선 | 브라우저 내 분석 | 결과 저장 범위와 품질지표 명시 |
| 접근통제 | 세션/권한 일부 | 권한표, 감사로그 접근통제 |
| 감사로그 | HMAC 감사로그 일부 | 실패 이벤트, 보존정책, export |
| 백업/복구 | 부족 | DB/파일/결과 ZIP 복구 시나리오 |
| 침투시험 | 미수행 | 외부 위탁 또는 내부 사전점검 계획 |

## 9. AI 성능평가와 임상자료

AI/STT/채점은 진단 또는 치료 결정을 자동 수행하지 않는다. 허가자료에서는 치료사 검토용 보조지표의 성능과 한계를 제시해야 한다.

| 평가 항목 | 필요 자료 | 현재 상태 |
| --- | --- | --- |
| STT | WER, CER, RTF, P95 latency, 오류 유형 | runner/계획 있음, locked set 부족 |
| 안면/시선 | 랜드마크 품질, 추적률, attention/off-task | 기능 일부 구현, 제출용 결과표 부족 |
| AAC | sequence→intent 결정성, unknown 처리 | 규칙 기반 구현, 임상성능 아님 |
| K-WAB/자가진단 | target/transcript/정오답/scoreReason | 저장 구조 보강 필요 |
| 노래훈련 | MIT/음악 기반 재활 보조 근거, 내부 전후 지표 | 클레임 잠금 필요 |
| gold label | SLP 라벨, 평가자 일치도, 눈가림 | 부족 |

## 14. 외부 확인 필요 항목

상담은 이미 끝났으므로 이 섹션은 “앞으로 식약처/RA/시험기관에 확인할 항목”으로만 본다.

| 우선순위 | 확인 질문 |
| --- | --- |
| Q1 | BrainFriends 사용목적이 독립형 디지털의료기기소프트웨어 2등급 가능성 기준에 부합하는가 |
| Q2 | 치료사 검토용 보조지표 + 경과 모니터링 표현이면 제25조 분석성능 검증으로 임상평가 자료 일부 갈음 가능성이 있는가 |
| Q3 | STT 서버 보안 프록시 구조를 인프라/위탁 서비스로 설명하는 방식이 적절한가 |
| Q4 | 전문가용 SW 표시가 필요한지, 필요하다면 치료사 IFU와 검토 로그 수준은 어디까지 요구되는가 |
| Q5 | 사용적합성 평가는 자체 수행 가능인지, 지정/위탁기관 수행을 권고하는지 |
| Q6 | DTx/치료효과 주장은 1차 허가와 분리해 후속 변경허가로 가는 것이 적절한가 |
