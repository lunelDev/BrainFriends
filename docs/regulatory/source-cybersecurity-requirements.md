목적: 브레인프렌즈의 웹 기반 SaMD 보안 요구사항을 식약처 사이버보안 가이드라인 표 2 기준으로 추적한다.

## 1. 적용 원칙

- 사이버보안 요구사항은 허가·심사 시 제출되어야 하는 최소 요구사항으로 본다.
- 제품 특성상 적용할 수 없는 항목은 `미적용 사유`를 위험관리문서, 사용자 설명서, 설계문서 등에 남긴다.
- 브레인프렌즈는 웹 기반 독립형 SaMD이므로 물리적 변조 방지, 부트 무결성처럼 SaMD 예외가 명시된 항목은 `미적용` 후보로 관리한다.
- 현재 `~77% 일치` 표현은 최종 합격이 아니라 내부 준비율이다. 외부 보안시험/증적 전에는 `완료`로 표현하지 않는다.

## 2. 사이버보안 특성 기재

| 항목 | 브레인프렌즈 현재 초안 | 보완 필요 |
| --- | --- | --- |
| 사용되는 통신 기술 | HTTPS 기반 웹 통신, 서버 API, 외부 STT API, DB/스토리지 연결 | 실제 배포 환경의 도메인, TLS 버전, 외부 서비스 목록 확정 |
| 사용 환경 | 병원 내/외 사용 가능. 환자는 가정, 치료사는 의료기관 또는 관리 환경에서 사용 | 사용환경별 권장 보안 설정 필요 |
| 공용 네트워크망 사용 여부 | 사용 가능성이 높음 | 공용망 사용 시 TLS, 인증, rate limit, 감사로그, DoS 대응 근거 필요 |
| 주요 데이터 | 계정정보, 환자 결과, transcript, 녹음 파일, 안면/시선 지표, 보호자 링크, 감사로그 | 저장 위치와 보존 기간 확정 필요 |

| DC-03 | 안전한 암호화 사용과 키 관리 | 비밀번호 hash, HMAC, 토큰, TLS, 저장 암호화에 적용 | 부분 | 알고리즘/키관리 정책 문서화 |

| RA-01 | DoS 방지 | 로그인, STT, AAC, 리포트, 결과 API rate limit에 적용 | 부분 | 필수 기능 정의와 부하/DoS 시험 필요 |
| RA-02 | 백업 | DB/스토리지/audit/result backup에 적용 | 부족 | 백업 범위, 주기, 복구시험 필요 |

| RA-05 | 불필요한 기능/포트/서비스 비활성화 | 서버 배포환경과 API surface에 적용 | 부족 | 포트/서비스 식별, 비활성화 증적 필요 |

## 7. 기존 문서와의 차이

기존 `cloud-and-data-transfer.md`는 STT/외부 전송과 일부 보안 정책 중심이다. 본 가이드라인 기준으로는 다음 항목이 추가로 필요하다.

- 전체 역할-권한 매트릭스
- API별 입력값 검증 표
- 감사로그 실패 대응
- 백업/복구 시험
- 운영환경 포트/서비스 점검
- TLS/암호화/키관리 증적
- 업데이트 진본성/무결성 검증
