# KTL 사전 문의 — 당사 확인받고자 하는 질문

## 목적

본 문서는 (주)골든브레인케어가 준비 중인 언어재활 지원 SaMD **"브레인프렌즈"** 에 대해, 귀 기관(KTL 의료기기평가센터)이 발급 가능한 공인성적서 범위·형식·절차를 사전에 확정하기 위한 질문 리스트입니다. 첨부의 제품 정의(01) · 준비 현황(02) 을 먼저 확인하신 뒤 아래 항목에 회신 부탁드립니다.

## 1. 수행 범위 — 반드시 확인

1-1. **소프트웨어 의료기기 평가** 영역에서 귀 기관이 발급 가능한 공인성적서는 다음 중 어디까지인가요?
   - [ ] SW V&V (IEC 62304 기반)
   - [ ] 사이버보안 (MFDS 가이드라인 / IEC 81001-5-1 기반)
   - [ ] AI 성능평가 (정량 지표 기반)
   - [ ] 사용적합성 (IEC 62366)
   - [ ] 위의 항목 중 귀 기관이 직접 수행하지 않는 부분은 어느 기관으로 안내 가능할지 추천 부탁드립니다.

1-2. 위 네 항목을 **한 건으로 통합 의뢰** 가능한지, 아니면 **각 항목별로 분리 의뢰** 해야 하는지 알려주세요.

## 2. 품목·등급 — 사전 의견 요청

2-1. 당사 사용목적 문구 (첨부 `01-product-definition.md` 7절) 기준으로, 본 제품이 의료기기법상 어떤 **품목군** 에 분류될 것으로 예상하시는지 의견 부탁드립니다.

2-2. **등급 (1 ~ 4 등급)** 을 사전 검토 단계에서 예측해 주실 수 있는지, 불가능하다면 어떤 추가 정보가 있어야 판단 가능한지 안내 부탁드립니다.

## 3. SW V&V — 기존 증적 적용 가능성

3-1. 당사는 현재 다음을 갖추고 있습니다 (첨부 `03-sw-vnv-outline.md`, `04-sw-vnv-current-report.md`).
   - 요구사항-시험-결과 추적성 매트릭스
   - deterministic 시험 12개 + 날짜별 실행 로그
   - 치료사 시스템 `/api/therapist/system/vnv-export` 기반 runtime evidence

3-2. **위 자료를 귀 기관 SW V&V 시험의 입력으로 인정**해 주실 수 있는지? 인정되지 않는다면 어떤 양식·증적 수준으로 재정리가 필요한지 안내 부탁드립니다.

3-3. 귀 기관이 사용하는 **SW V&V 결과서 표준 양식 (한글/엑셀/PDF)** 이 있다면 샘플 한 부 부탁드립니다.

## 4. 사이버보안 — 기준 및 추가 요구

4-1. 당사는 현재 다음을 갖추고 있습니다 (첨부 `05-cybersecurity-report.md`, `06-cybersecurity-inventory.md`).
   - 브라우저 저장 최소화 및 민감 raw 데이터 제거
   - `src/lib/security/transientStepStorage.ts` 기반 세션 범위 저장 정책
   - `/api/security-audit` 기반 보안 이벤트 기록

4-2. 귀 기관이 사이버보안 시험 시 **추가로 요구하는 절차서·정책 문서** (예: 위협모델링, 침해 대응 절차, SBOM) 가 있는지 목록으로 회신 부탁드립니다.

4-3. **IEC 81001-5-1 / MFDS 디지털의료기기 사이버보안 허가 가이드라인** 중 어느 기준으로 시험이 진행되는지 확인 부탁드립니다.

## 5. AI 성능평가 — 데이터·라벨·프로토콜

5-1. 당사는 현재 다음 구조를 갖추고 있습니다 (첨부 `07-ai-evaluation-outline.md`, `08-ai-evaluation-current-report.md`).
   - `measured` 품질 샘플만 평가셋으로 분리
   - `src/lib/server/evaluationSamplesDb.ts` 기반 DB 우선 저장
   - 버전 / 모델 / 데이터셋 3축 비교 모니터링
   - `/api/therapist/system/ai-evaluation-export` 제출형 export 경로

5-2. 귀 기관 AI 성능평가 수행 시 요구되는 **최소 샘플 수·정답 라벨 정의·결과표 지표** (예: accuracy / F1 / bias 분석 등) 를 안내 부탁드립니다.

5-3. **음성 도메인 (한국어 aphasia / dysarthria 환자)** 평가셋에 대한 귀 기관의 레퍼런스 또는 권고 기준이 있는지, 있다면 공유 부탁드립니다.

## 6. 음향 분석 (Parselmouth / Praat) — 결정론 시험

6-1. 당사는 `voice-analysis-service/` 에 Parselmouth 기반 음향 분석 엔진을 자체 구축했습니다 (첨부 `02-readiness-summary.md` 및 `11-CHANGELOG.md` 참조). 구현 지표:
   - REQ-ACOUSTIC-001~004 (F0 / Intensity / Voicing Ratio / Duration)
   - REQ-ACOUSTIC-010~012 (Jitter local / Shimmer local / HNR)
   - REQ-ACOUSTIC-030 (measurement_quality 게이트)
   - REQ-ACOUSTIC-050 (version_snapshot)

6-2. 귀 기관이 **Parselmouth / Praat 기반 음향 지표의 결정론 재현 시험** 을 수행 가능한지? 가능하다면 **허용오차 표기 형식** (예: 명세값 ±2% / ±5%) 을 어떻게 결과서에 반영해 주시는지 안내 부탁드립니다.

6-3. 현재 REQ-ACOUSTIC-020/021 (Formant F1/F2) 는 미구현 상태입니다. **구현 후 시험 의뢰 시점에 맞춰 후속 의뢰가 가능한지**, 아니면 최초 의뢰 시점에 전체 지표를 동시 제출해야 하는지 확인 부탁드립니다.

## 7. 제출 형식 — 양식·파일 포맷

7-1. 최종 제출물의 **양식 기준** 을 알려주세요.
   - 한글(HWP) / Word(DOCX) / PDF 중 어느 것이 필수·권장인지
   - 추적성 매트릭스 등 표 양식의 엑셀 원본이 필요한지

7-2. **필수 부록** 리스트 (제조사 품질관리시스템 증빙, 위험관리 파일, 소프트웨어 안전 등급 분류 문서 등) 가 있다면 알려주세요.

## 8. 일정·비용·절차

8-1. 사전 검토 → 의뢰 접수 → 시험 수행 → 성적서 발급까지의 **표준 절차와 각 단계별 소요 기간** 을 알려주세요.

8-2. 본 제품 범위 기준 예상 **시험 비용 개략치** (항목별 또는 총액) 를 제시 가능하신지, 불가능하다면 견적을 받기 위해 필요한 입력 정보는 무엇인지 안내 부탁드립니다.

8-3. 당사가 다음 단계로 넘어가기 위해 먼저 확보해야 하는 최우선 자료는 무엇인지 말씀 부탁드립니다.

## 9. 회신 요청

본 문서 기준으로 **항목별 답변** 또는 **대면·화상 사전 검토 미팅** 중 편하신 방식으로 회신 부탁드립니다. 미팅이 필요하다면 당사 측에서 일정을 맞추겠습니다.

감사합니다.
